# Agent 协同与二维线弹性演示文档体系实施计划

> **面向代理执行者：** 必须使用 `executing-plans` 或 `subagent-driven-development` 按任务顺序实施；步骤使用复选框跟踪。用户此前选择在 `main` 分支内联执行时，不创建额外分支或 worktree。

**目标：** 把现有完整但分散的仓库文档整理成适合新 Agent 快速接管、面向非专业受众演示二维线弹性价值、同时支持技术人员继续开发新 PDE 的分层文档体系。

**架构：** 仓库事实继续由配置、操作指南和运行报告负责；新增文档地图、能力状态、Agent 协作指南和演示手册只负责路由与解释。`AGENTS.md` 提供最小必读顺序，文档测试负责入口文件、关键语义和本地链接完整性；未来 Skill 只形成内容草案，不在本阶段安装或发布。

**技术栈：** 中文 Markdown、pytest、Python `pathlib`、正则表达式、uv、Ruff、现有 `surrogate-loop` CLI。

## 全局约束

- 所有用户可读新增文档使用中文。
- 设计规格和实施计划直接保存在 `docs/` 根目录，不创建 `docs/superpowers/`。
- 现有设计规格和实施计划不移动、不重命名，避免破坏历史链接。
- `README.md` 继续面向普通用户，不复制完整演示剧本或大型状态表。
- `runs/`、训练数据、模型和本地报告不提交 Git。
- 不修改模型、求解器、配置门槛、数据划分、CLI 行为或 Full 状态。
- 动态实测数值只在 `docs/当前能力与状态.md` 和对应操作指南中维护。
- 演示手册必须区分假求解 E2E、真实 FEniCSx E2E、calibration、Smoke 和 Full。
- 二维线弹性当前只完成 Smoke；文档不得称其为 accepted Full 或允许正常可信推理加载。
- 从头运行发生阶段复用时，Agent 必须明确说明，不能称为完全从零运行。
- Full 未经用户新的明确确认不得启动。
- 文档测试先失败，再创建或修改对应文档。
- 每个任务结束运行目标测试；最终运行 Ruff、完整 pytest、求解器测试和真实微型 E2E。

```python
MARKDOWN_LINK = re.compile(r"\[[^\]]+\]\(([^)]+)\)")


def test_local_markdown_links_resolve() -> None:
    broken: list[str] = []
    documents = [ROOT / "AGENTS.md", ROOT / "README.md", *sorted((ROOT / "docs").rglob("*.md"))]
    for document in documents:
        content = document.read_text(encoding="utf-8")
        for raw_target in MARKDOWN_LINK.findall(content):
            target = raw_target.strip().strip("<>")
            if not target or target.startswith("#") or "://" in target:
                continue
            relative = target.split("#", 1)[0]
            if not relative:
                continue
            candidate = (document.parent / relative).resolve()
            if not candidate.exists():
                broken.append(
                    f"{document.relative_to(ROOT)} -> {target}"
                )
    assert broken == []
```

该检查只验证提交到仓库的相对链接。`runs/` 路径在演示手册中必须使用代码格式，因此不会被误认为全新克隆应存在的文档链接。
