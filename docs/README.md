# 项目文档地图

本页是仓库文档的导航入口。它帮助新 Agent 和使用者按当前任务读取最少且足够的材料，避免把历史实施计划误当成当前运行状态。

## 新 Agent 从这里开始

最小阅读顺序：

1. [仓库工作规则](../AGENTS.md)
2. 本文档
3. [当前能力与状态](当前能力与状态.md)
4. 查看 `git status --short` 和最近提交
5. 按下方任务路由继续读取

## 按任务选择阅读路径

### 演示

管理层工作汇报优先读取[第 01 期代理模型训练闭环周报](周报/2026-07-17-第01期-代理模型训练闭环周报.md)。技术演示再读[当前能力与状态](当前能力与状态.md)，确认可展示的证据等级；当前推荐以二维线弹性悬臂梁为主线，从[演示入口](demos/README.md)选择快速展示或从头运行，并按[二维线弹性演示手册](demos/二维线弹性演示手册.md)讲解公式、六阶段进度和证据边界。

### 运行已有闭环

选择对应操作指南：

- [标量 ODE 闭环](guides/标量闭环操作指南.md)
- [一维热传导闭环](guides/一维热传导闭环操作指南.md)
- [二维线弹性闭环](guides/二维线弹性闭环操作指南.md)
- [Windows 跨机迁移指南](guides/Windows跨机迁移指南.md)：在另一台 Windows 11 + NVIDIA GPU 电脑上恢复完整链路或导入 accepted 运行。

先执行配置校验，再根据指南判断是 Smoke、Full 还是只读报告。二维线弹性 Full 必须重新获得用户明确授权。

### 诊断

先读对应操作指南中的故障处理，再查看本次运行目录里的状态、错误、摘要和诊断文件。环境类问题从[环境与验证](guides/环境与验证.md)开始；不要通过删除失败样本、绕过摘要或放宽 Full 门槛来消除失败。

### 开发新 PDE

先确认问题是否超出现有白名单模板，再走“中文规格评审 → 中文实施计划 → 测试驱动实现 → 分级验收”。可参考[二维线弹性闭环设计](2026-07-16-二维线弹性神经算子闭环设计.md)理解求解环境、训练环境和协议的分层方式，但不要直接复制物理假设。

## 文档事实来源

- 配置文件定义可执行的输入、预算、数据划分和验收合同。
- 操作指南定义当前命令、环境、产物和故障处理方式。
- [当前能力与状态](当前能力与状态.md)汇总最近里程碑与推荐演示证据。
- 单次运行目录中的报告、状态和摘要定义那一次运行的实际结果。
- 设计规格解释稳定架构和科学边界；历史实施计划只用于追溯。
