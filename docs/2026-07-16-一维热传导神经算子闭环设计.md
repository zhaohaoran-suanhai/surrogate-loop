# 一维热传导神经算子闭环设计

## 1. 建设目标

在已经完成的标量代理模型闭环之上，建立一个能够学习完整时空场的第二阶段闭环，用来验证从数值算例生成、神经算子训练、独立验收到可信推理的完整技术路线。

本阶段固定支持一维瞬态热传导问题：

\[
\frac{\partial u}{\partial t}=\alpha\frac{\partial^2u}{\partial x^2},
\qquad x,t\in[0,1],
\]

\[
u(0,t)=u(1,t)=0,
\]

\[
u(x,0)=A\sin(\pi x)+B\sin(2\pi x),
\]

其中

\[
\alpha\in[0.05,0.2],\qquad
A\in[0.8,1.2],\qquad
B\in[-0.3,0.3].
\]

学习合同为：

\[
\mathcal G:(\alpha,A,B)\longmapsto u(\cdot,\cdot).
\]

DeepONet 的 Branch 输入为 `(alpha, A, B)`，Trunk 输入为查询坐标 `(x, t)`，输出为 `u(x,t)`。本阶段的重点是验证神经算子闭环，而不是构建任意 PDE 解析器或通用仿真平台。

## 2. 方案选择

本阶段采用 PyTorch DeepONet 作为主模型，同时实现 POD/GPR 作为诊断基线。

- DeepONet 与后续二维场、几何条件和 CFD 代理模型的技术路线一致。
- POD/GPR 用于判断数据生成、划分、降维和评估链路是否正常。
- POD/GPR 通过不能替代 DeepONet 验收；DeepONet 未达到门槛时，本次神经算子闭环仍判为未通过。
- 第一版不加入 PDE 自动微分残差、FNO、混合精度、自动超参数搜索或分布式训练，避免在闭环尚未稳定时同时引入过多变量。

## 3. 范围

### 3.1 范围内

- 固定的一维热传导 PDE 模板和受约束参数域。
- 独立解析解和 Crank–Nicolson 数值求解器。
- 确定性 Latin Hypercube 参数采样。
- 按完整 PDE 算例隔离训练、验证和测试集。
- 完整时空场数据生成、缓存和身份记录。
- POD/GPR 基线。
- PyTorch DeepONet、早停、最佳检查点和训练恢复。
- 场误差、初值误差、边界误差和域外拒绝。
- Smoke 与 Full 两档可复现实验配置。
- 点预测、规则网格完整场预测和运行产物导出。
- CUDA 优先、CPU 回退的运行策略。

### 3.2 范围外

- 任意 PDE 字符串解析和动态执行生成代码。
- 非结构网格、二维或三维 PDE、复杂几何和真实 CFD 求解器。
- PINN/PINO 物理残差训练、FNO 和其他神经算子架构比较。
- 自动调参、多 GPU、分布式训练、在线服务和生产部署。
- 域外外推承诺。
- 把解析解直接用作训练标签。解析解只用于校验数值求解器和评估科学误差。

## 4. 架构原则

采用“共享入口、领域隔离”的方式接入现有仓库：

- 保持已经可运行的标量闭环及其命令兼容。
- 新增 `surrogate_loop.operator` 子系统，不把 PyTorch 强制加入标量闭环的基础依赖。
- PDE、数据、模型、训练、评价、产物和推理职责分离。
- 每个运行目录都能独立回答“用什么配置、什么数据、什么环境、什么模型、为何通过或拒绝”。

目标目录如下：

```text
src/surrogate_loop/
├── 原有标量闭环模块
└── operator/
    ├── config.py
    ├── pipeline.py
    ├── artifacts.py

    └── heat1d/

        ├── analytical.py
        ├── solver.py
        ├── dataset.py
        ├── pod_gpr.py
        ├── deeponet.py

tests/

```

`examples/` 只保存配置和中文算例说明，核心实现只能放在 `src/surrogate_loop/`。生成的数据、模型和图表保存在 `runs/`，不提交 Git。

## 5. 命令边界

现有标量命令保持不变。神经算子使用独立命令组：

```text
surrogate-loop operator validate --config <path>
surrogate-loop operator run --config <path>
surrogate-loop operator report --run-dir <path>
surrogate-loop operator predict --run-dir <path> --alpha <value> --a <value> --b <value>
```

`predict` 支持指定 `(x,t)` 的单点预测和指定规则网格的完整场预测。输入参数必须有限且位于训练参数域内，域外请求在加载模型和执行推理前拒绝。

第一版仍由 Codex 把用户已经确认的中文需求转换为白名单 JSON 配置。运行时 CLI 不解析任意自然语言或任意方程。

## 6. 科学计算设计

### 6.1 解析解

独立解析解为：

\[
u(x,t)=A e^{-\alpha\pi^2t}\sin(\pi x)
+B e^{-4\alpha\pi^2t}\sin(2\pi x).
\]

解析解不参与训练标签生成。它用于验证数值求解器、构造科学单元测试，并在最终报告中区分“代理模型误差”和“数值标签误差”。

### 6.2 数值求解器

使用 Crank–Nicolson 有限差分格式：

- 空间使用均匀网格和二阶中心差分。
- 时间使用 Crank–Nicolson 二阶隐式推进。
- 边界值在每个时间步显式保持为零。
- 初值由参数 `(A,B)` 在网格上直接计算。
- 线性系统使用 SciPy 的确定性带状或三对角求解接口。

求解器必须在生成训练数据前通过解析解校验。输出非有限、线性求解失败、边界误差或解析校验超限都会终止运行，不启动模型训练。

### 6.3 参数采样与数据表示

使用 SciPy `qmc.LatinHypercube` 在三维参数域确定性采样。随机种子进入配置和运行清单。

每个算例保存：

- 参数向量 `(alpha,A,B)`；
- 空间坐标 `x`；
- 时间坐标 `t`；
- 数值温度场 `U[time, space]`；
- 数值解与解析解的校验摘要。

训练、验证和测试必须按完整参数算例划分。同一个 `(alpha,A,B)` 的任何时空点不能跨集合。所有参数、坐标和目标归一化统计量只能由训练集拟合。

## 7. 模型设计

### 7.1 POD/GPR 基线

- 将每个训练温度场展平为一个向量。
- POD 只在训练集拟合。
- 保留累计能量达到 `99.9%` 的模态，最多 32 个。
- 使用 GPR 从 `(alpha,A,B)` 分别预测 POD 系数。
- 在验证集和测试集上重构完整温度场。
- 与 DeepONet 使用完全相同的数据划分和评价指标。

### 7.2 DeepONet

默认结构：

- Branch：`3 -> 128 -> 128 -> 128 -> 128`；
- Trunk：`2 -> 128 -> 128 -> 128 -> 128`；
- 隐层激活：`tanh`；
- 初始化：Xavier；
- 输出：Branch 和 Trunk 的 128 维表示内积，再加可训练偏置；
- 优化器：Adam；
- 初始学习率：`1e-3`；
- 学习率调度：验证损失停滞后下降；
- 损失：使用训练集统计量归一化后的监督 MSE。

网络原始输出经过问题专用的硬约束层：以解析初值为基底，并将网络修正项乘以 `4x(1-x)t`。因此在浮点误差范围内严格满足 `u(x,0)=u_0(x)` 和两端零边界；这里仅使用已知初边值结构，不把解析时间演化解作为标签或模型输入。

训练批次采用“算例批量 × 共享查询坐标”的笛卡尔积计算，避免把全部算例和全部网格点一次性展开。验证损失决定早停点和最佳检查点；测试集不参与结构选择、学习率调整和早停。

模型产物只保存 `state_dict`、网络结构配置、归一化统计量和数据身份，不保存不可审计的完整 Python 模型对象。

## 8. 实验规模

| 项目 | Smoke | Full |
|---|---:|---:|
| 空间网格 `nx` | 65 | 129 |
| 时间网格 `nt` | 51 | 101 |
| 训练算例 | 64 | 512 |
| 验证算例 | 16 | 96 |
| 测试算例 | 16 | 128 |
| 最大 epoch | 1500 | 600 |
| early stopping patience | 120 | 60 |
| 算例批量 | 16 | 32 |
| 每批查询坐标 | 256 | 512 |
| 数值精度 | FP32 | FP32 |

1500/600 epoch 分别是 Smoke/Full 配置上限，而不是必须执行的固定轮数。Smoke 每轮 4 个参数批次，Full 每轮 16 个参数批次；两档的优化步数不能直接用 epoch 数比较。训练使用早停并保存最佳检查点。

在当前 RTX 4060 Laptop 8 GB、32 GB 内存环境上，正式 Smoke/Full 的 DeepONet 训练分别实测约 86/88 秒，峰值 CUDA 已分配显存分别约 19/20 MiB，并都在独立测试集通过全部验收门槛。Full 仍保留 60 分钟软预算；达到软预算时保存检查点和诊断信息，不无限运行，也不通过降低验收标准换取通过状态。上述时间只适用于当前固定问题、软件版本和硬件环境。

## 9. 评价与验收

### 9.1 求解器验收

- 所有输出有限。
- 零边界最大绝对误差不超过 `1e-10`。
- Full 网格下，逐算例解析解相对 L2 误差的 95 分位不超过 `1e-3`。
- Smoke 使用单独配置的、与较粗网格相符的求解器阈值。
- 求解器未通过时，运行状态为 `failed`，禁止训练。

### 9.2 DeepONet 验收

独立测试集必须同时满足：

- 中位逐算例相对 L2 误差不超过 `2%`；
- 95 分位逐算例相对 L2 误差不超过 `5%`；
- 最坏算例相对 L2 误差不超过 `10%`；
- 初始场相对 L2 误差不超过 `5%`；
- 边界最大绝对误差不超过 `1e-2`；
- 所有预测有限；
- 域外和非有限参数被拒绝。

POD/GPR 的相同指标进入比较报告，但不替代 DeepONet 的绝对门槛。POD/GPR 通过而 DeepONet 失败时，报告应明确结论为“数据与评估链路正常，神经算子未达标”。

推理耗时、数值求解耗时和加速比只记录，不作为第一版硬门槛。该 PDE 求解器规模很小，GPU 初始化和进程启动可能掩盖模型推理优势。

## 10. 运行产物

每次运行创建唯一目录，至少保存：

- 原始请求记录和校验后的配置快照；
- Python、PyTorch、CUDA、GPU、设备和随机种子信息；
- 参数样本、数据划分成员和数据哈希；
- 数值求解器解析校验摘要；
- 训练集归一化统计量；
- POD 模态、GPR 基线和指标；
- DeepONet 最佳 `state_dict` 和网络结构配置；
- 训练历史、学习率和早停原因；
- 验证与测试指标；
- 代表性算例的真值场、预测场和误差场图；
- 模型卡、最终状态和 SHA-256 清单。

只有状态为 `accepted` 的 DeepONet 运行可以通过正常推理接口加载。`rejected` 和 `failed` 运行保留诊断产物，但不能作为已验收模型使用。

## 11. 环境方案

保持 Python `3.11.x` 和 `uv`。PyTorch 放入独立的 `operator` 可选依赖组：

- 固定 `torch==2.9.0`；
- Windows 使用 PyTorch 官方 CUDA 12.6 wheel；
- 安装入口为 `uv sync --extra operator --dev`；
- 不要求从源码构建 PyTorch；
- 运行设备默认 `auto`，CUDA 可用时使用 GPU，否则退回 CPU；
- 第一版统一使用 FP32，不启用自动混合精度。

环境安装后必须执行以下冒烟验证：

1. 导入 PyTorch 并记录版本。
2. 检查 `torch.cuda.is_available()`。
3. 在实际设备上完成小型张量前向和反向传播。
4. 验证显式 CPU 模式可运行。
5. 运行现有标量闭环测试，确认新增依赖没有破坏已有功能。

CUDA 版 PyTorch、安装目录和 uv 缓存预计占用 3 到 6 GB。生成数据和单次 Full 运行产物预计低于 1 GB。应避免长期累积无用运行目录和下载缓存。

## 12. 失败处理

- 配置无效：在数值求解和训练前拒绝。
- PyTorch 可选依赖缺失：给出准确安装命令，不影响标量闭环。
- CUDA 初始化失败：记录原因并退回 CPU。
- CUDA 显存不足：保存失败清单并停止，不静默改变网络、批量或验收标准。
- 求解器解析校验失败：标记 `failed`，禁止训练。
- 训练出现 NaN/Inf：保存最后正常检查点和诊断，标记 `failed`。
- 达到软时间预算：保存最佳检查点和当前状态，明确标记预算终止，不能伪装为完成。
- DeepONet 未达门槛：标记 `rejected` 并保留完整诊断。
- 产物哈希不匹配、结构配置不一致或域外输入：拒绝模型加载或推理。

## 13. 测试策略

实现遵循测试驱动开发，先写失败测试，再写最小实现。

### 13.1 单元测试

- 配置字段、范围、未知字段和 Smoke/Full 预算校验。
- 解析解满足 PDE、边界和初值条件。
- Crank–Nicolson 对解析解达到规定收敛误差。
- 固定种子产生相同参数样本。
- 算例级数据划分互不相交。
- 归一化器只使用训练集。
- DeepONet 张量形状、前向传播、梯度和保存重载。
- 域外、NaN 和无穷参数拒绝。

### 13.2 集成测试

- 求解器未通过时训练器不会启动。
- POD/GPR 和 DeepONet 使用相同分区。
- 验证集控制早停，测试集只最终评估一次。
- 最佳检查点保存前后预测一致。
- CUDA 不可用时 CPU 回退。
- 运行清单损坏时模型加载被拒绝。

### 13.3 端到端测试

- 使用小于正式 Smoke 预算的测试专用配置运行完整 CLI。
- 生成所有必需产物和最终状态。
- 在新进程中加载已验收模型并完成单点和完整场预测。
- 域外推理失败且退出码稳定。
- 原有标量端到端测试继续通过。
