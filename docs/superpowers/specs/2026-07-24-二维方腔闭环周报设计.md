# 二维方腔闭环周报设计

## 1. 目标

在现有项目周报体系中新增第 02 期周报，向项目负责人、管理层和技术复核人员展示二维顶盖驱动方腔代理模型从真实 Fluent 数据生成、Smoke 开发验证、Full 封存验收到域内推理演示的完整闭环与实际效果。

本次更新还要同步稳定文档中的能力状态，消除“方腔尚未完成 Full”的过时描述，并让关键可视化随 Git 分发，不依赖被忽略的本地 `runs/` 目录。

## 2. 发布位置

文档先在 `feature/cavity2d-surrogate` 分支完成，与方腔功能代码和证据摘要一起接受检查，随后整体合入 `main`。这样可以避免主分支文档先于实际能力发布。

## 3. 内容定位

采用“管理层可读、技术证据可追溯”的平衡方案：

- 正文能够独立说明问题、闭环、效果、成本和边界。
- 关键指标来自已完成的 Full 运行和 Re=310.5 独立演示。
- 不在周报正文展开所有批次级日志和完整哈希，但保留本地证据路径、运行身份和复核入口。
- 不把域内插值结果表述为任意 Navier–Stokes、任意几何或训练域外求解能力。

## 5. 周报结构

第 02 期周报按以下顺序组织：

1. Executive Summary。
2. 首版问题合同与能力边界。
3. Fluent—代理模型训练闭环架构。
4. 从 vertical、calibration、Smoke 到 Full 的里程碑。
5. Full 120 工况封存验收结果。
6. Re=310.5 未见工况的 Fluent 与代理模型对比。
7. 三组可视化。

10. 当前结论、限制和下一阶段建议。

### Re=310.5 演示

Re=310.5 不在 Full 的训练、验证或封存测试样本中，但位于冻结训练域 `[10,400]` 内。

- Fluent：1291 次迭代，3550 个单元。
- 速度场相对 L2：`0.159169%`
- 去均值压力场相对 L2：`4.045223%`
- 合并中心线速度相对 L2：`0.329163%`
- 主涡中心位置误差：`0`
- Fluent 单样本墙钟：`39.0594406 s`
- 代理冷启动端到端中位墙钟：`1.8051988 s`，约 `21.64×`
- 模型预加载后纯推理平均墙钟：`0.0001819358 s`，约 `214688×`

冷启动和预加载计时必须分开呈现。后者只代表本机纯 `model.predict`，不是生产端到端基准。
