# 二维方腔闭环周报 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 新增第 02 期二维方腔代理模型闭环周报、纳入三张 Re=310.5 对比图片，并把稳定文档中的方腔能力状态更新为真实 Full `accepted`。

**Architecture:** 周报正文负责独立说明闭环、Full 指标、Re=310.5 演示、性能口径和能力边界；图片复制到受 Git 管理的周报资产目录。周报索引、文档地图、当前能力状态和方腔算例 README 共同形成从管理摘要到技术操作合同的导航链。

**Tech Stack:** Markdown、PNG、Git、PowerShell、Python/pytest、Ruff

## Global Constraints

- 文档先在 `feature/cavity2d-surrogate` 分支完成，随后随方腔功能整体合入 `main`。
- 关键事实只采用 `cavity2d-full-20260723T185548Z-8cb01c1e` 和 `cavity2d-demo-re310p5-20260724` 的已存在证据。
- 不重新启动 Fluent、训练模型或消费 sealed-test。
- 图片必须复制到 `docs/周报/assets/2026-07-24-cavity2d/`，不能依赖被 Git 忽略的 `runs/`。
- Full 历史字段 `cpu_speedup` 按墙钟比解释，不表述为累计 CPU time。
- 冷启动端到端和模型预加载后的纯推理必须分开呈现。
- 能力边界固定为单位方腔、固定顶盖速度、稳态不可压层流、`Re ∈ [10,400]` 和固定网格。
- 不宣称支持任意 Navier–Stokes、任意几何、瞬态、湍流或训练域外问题。
