# 标量代理模型闭环操作指南

## 自然语言入口

第一版由 Codex 负责理解用户意图。例如：

> 用强迫反应 ODE 的冒烟配置生成数据，比较 PRS、GPR 和 MLP，保存最佳模型并预测 gamma=0.35。

Codex 不生成并执行任意科学代码，而是选择白名单配置并调用确定性 CLI。

## 校验和训练

```powershell
uv run surrogate-loop validate --config examples/forced_reaction_scalar/smoke.json
uv run surrogate-loop run --config examples/forced_reaction_scalar/smoke.json --smoke --request "冒烟训练并选择最佳代理模型"
```

`run` 返回 JSON，其中 `run_dir` 是本次运行目录。目录内包含配置、数据、分区、验证指标、测试指标、最佳模型、模型卡、预测图和带 SHA-256 的清单。

## 报告和推理

```powershell
uv run surrogate-loop report --run-dir runs/运行标识
uv run surrogate-loop predict --run-dir runs/运行标识 --gamma 0.35
```

当 `gamma` 超出 `[-1,1]`，或输入为 NaN/无穷值时，命令以退出码 2 拒绝预测。模型或配置文件被修改后，清单哈希校验也会拒绝加载。

## 闭环边界

当前闭环证明的是“自然语言能够安全驱动确定、可验证、可复现的代理模型工具链”。它尚不是通用 PDE 平台，也不支持任意方程、FEniCSx、DeepONet 或运行时多 Agent。
