# 环境与验证

## 创建环境

仓库固定使用 Python 3.11 和 uv：

```powershell
uv python pin 3.11
uv sync --all-groups
```

uv 会在仓库内创建 `.venv`，并按照 `uv.lock` 同步依赖。不要向系统 Python 安装本项目依赖。

## 神经算子 GPU 环境

神经算子使用独立可选依赖，不影响只运行标量闭环的用户：

```powershell
uv sync --extra operator --all-groups
uv run python -c "import torch; print(torch.__version__, torch.version.cuda, torch.cuda.is_available())"
```

本仓库锁定 `torch==2.9.0` 和官方 CUDA 12.6 wheel。本机预期输出包含：

```text
2.9.0+cu126 12.6 True
```

首次下载约 2.4 GiB；受网络和 Windows Defender 大文件扫描影响，本机首次准备耗时约 28 分钟。后续同步会复用 uv 缓存。

验证 GPU 前后向：

```powershell
uv run python -c "import torch; x=torch.randn(128,128,device='cuda',requires_grad=True); y=x.square().mean(); y.backward(); print(torch.cuda.get_device_name(0), torch.isfinite(x.grad).all().item())"
```

代码运行策略为 `auto`：CUDA 可用时选择 GPU，否则退回 CPU。若配置明确写成 `device: cuda` 而 CUDA 不可用，运行会失败而不是静默回退。

## 验证骨架

```powershell
uv run surrogate-loop --help
uv run python -m surrogate_loop --version
uv run ruff check .
uv run pytest
```

预期版本输出为：

```text
surrogate-loop 0.1.0
```

完整质量门在安装 operator extra 后运行：

```powershell
uv run ruff check .
uv run pytest
```

运行 Task 9 检查点时，全量 73 项测试通过，其中包含标量和热传导两个子进程级端到端闭环。
