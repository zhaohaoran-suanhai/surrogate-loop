# 环境与验证

## 创建环境

仓库固定使用 Python 3.11 和 uv：

```powershell
uv python pin 3.11
uv sync --all-groups
```

uv 会在仓库内创建 `.venv`，并按照 `uv.lock` 同步依赖。不要向系统 Python 安装本项目依赖。

## 神经算子 GPU 环境

神经算子使用独立可选依赖，不影响只运行标量闭环的用户：

```powershell
uv sync --extra operator --all-groups
uv run python -c "import torch; print(torch.__version__, torch.version.cuda, torch.cuda.is_available())"
```

本仓库锁定 `torch==2.9.0` 和官方 CUDA 12.6 wheel。本机预期输出包含：

```text
2.9.0+cu126 12.6 True
```

首次下载约 2.4 GiB；受网络和 Windows Defender 大文件扫描影响，本机首次准备耗时约 28 分钟。后续同步会复用 uv 缓存。

验证 GPU 前后向：

```powershell
uv run python -c "import torch; x=torch.randn(128,128,device='cuda',requires_grad=True); y=x.square().mean(); y.backward(); print(torch.cuda.get_device_name(0), torch.isfinite(x.grad).all().item())"
```

代码运行策略为 `auto`：CUDA 可用时选择 GPU，否则退回 CPU。若配置明确写成 `device: cuda` 而 CUDA 不可用，运行会失败而不是静默回退。

## 验证骨架

```powershell
uv run surrogate-loop --help
uv run python -m surrogate_loop --version
uv run ruff check .
uv run pytest
```

预期版本输出为：

```text
surrogate-loop 0.1.0
```

完整质量门在安装 operator extra 后运行：

```powershell
uv run ruff check .
uv run pytest
```

运行 Task 9 检查点时，全量 73 项测试通过，其中包含标量和热传导两个子进程级端到端闭环。

## 二维线弹性 FEniCSx 求解环境

二维弹性闭环把有限元求解与 PyTorch 训练隔离。Windows 侧需要 Miniforge 和 Visual Studio 2022 Build Tools；Build Tools 中安装“使用 C++ 的桌面开发”、MSVC 和 Windows SDK。

```powershell
conda env create -f environments/fenicsx-0.11.yml
uv run surrogate-loop elasticity2d doctor
```

求解环境固定名为 `surrogate-loop-fenicsx-0.11`，使用 DOLFINx 0.11、PyAMG 和 SciPy，不依赖 `petsc4py`。训练环境仍通过以下命令准备：

```powershell
uv sync --extra operator --all-groups
```

显式验证求解侧和跨环境协议：

```powershell
conda run -n surrogate-loop-fenicsx-0.11 python -m pytest tests/solver/elasticity2d -v
$env:SURROGATE_LOOP_RUN_FENICSX_E2E='1'
uv run pytest tests/e2e/test_elasticity2d_fenicsx_loop.py -v
Remove-Item Env:SURROGATE_LOOP_RUN_FENICSX_E2E
```

日常 `uv run pytest` 会跳过真实 FEniCSx E2E，因此普通测试不能被误读为有限元科学验证。完整配置、校准、Smoke 和排错步骤见 [二维线弹性闭环操作指南](二维线弹性闭环操作指南.md)。
