# 二维线弹性闭环操作指南

## 1. 闭环边界

第一版实现固定二维平面应力悬臂梁模板：

```text
自然语言需求
→ 人工确认后的白名单 JSON
→ FEniCSx 数值求解与数据生成
→ POD-RBF 诊断基线和 Vector DeepONet 训练
→ 开发评价或一次性封存验收
→ 已验收模型的快速位移场推理
```

Codex 负责把用户已确认的自然语言需求映射为仓库中受审查的结构化 JSON。运行时 CLI 不调用 LLM、不解析任意方程、不动态导入配置指定的代码，也不执行自然语言生成的代码。当前不是通用 PDE 编译器。

模型输入为 `(E, nu, P, theta, y0, w)`，输出为查询坐标上的 `(u_x, u_y)`。模型利用线弹性比例关系 `P/E` 和左端固支边界。第一版不把应力、应变或 von Mises 作为网络正式输出，也不承诺由网络位移导数得到的应力具有工程验收精度。

## 2. 双环境架构

训练侧使用仓库的 Python 3.11、uv、PyTorch 环境；求解侧使用独立的 `surrogate-loop-fenicsx-0.11` Conda 环境。两端只交换固定版本的 JSON/NPZ 文件和 SHA-256 摘要。

Windows 原生 FEniCSx 包需要 64 位 Windows、Miniforge，以及包含 MSVC C++ 编译器和 Windows SDK 的 Visual Studio 2022 Build Tools。安装 Build Tools 时选择“使用 C++ 的桌面开发”。本实现采用串行 PyAMG/SciPy，不依赖 Windows 包未提供的 `petsc4py`。

```powershell
uv python pin 3.11
uv sync --extra operator --all-groups
conda env create -f environments/fenicsx-0.11.yml
# 环境已存在时：
conda env update -n surrogate-loop-fenicsx-0.11 -f environments/fenicsx-0.11.yml --prune
```

两个环境不要合并。`dolfinx` 不安装到 `.venv`，PyTorch 训练依赖也不要求安装到 FEniCSx 环境。

## 3. 环境诊断和验证

```powershell
uv run surrogate-loop elasticity2d doctor
uv run surrogate-loop elasticity2d validate --config examples/elasticity_2d_cantilever/calibration.json
```

`doctor` 必须报告 `dolfinx 0.11.x`、PyAMG、SciPy 和 `petsc4py_available=false`。Windows 下外部进程强制使用 UTF-8，避免 `conda run` 在中文系统代码页上转发输出失败。

普通回归测试不要求 FEniCSx：

```powershell
uv run ruff check .
uv run pytest
```

求解器科学测试与真实微型跨环境测试需要显式运行：

```powershell
conda run -n surrogate-loop-fenicsx-0.11 python -m pytest tests/solver/elasticity2d -v
$env:SURROGATE_LOOP_RUN_FENICSX_E2E='1'
uv run pytest tests/e2e/test_elasticity2d_fenicsx_loop.py -v
Remove-Item Env:SURROGATE_LOOP_RUN_FENICSX_E2E
```

`tests/e2e/test_elasticity2d_loop.py` 使用假求解适配器，只证明 `validate → run → report → predict` 编排、保护机制和兼容性。它不能证明 FEniCSx 方程、物理门禁或科学正确性。只有求解器科学测试、显式真实 E2E、校准和正式数据清单才是 FEniCSx 证据。

## 4. 校准

先执行校准，不要直接生成 Smoke 或 Full 数据：

```powershell
uv run surrogate-loop elasticity2d calibrate --config examples/elasticity_2d_cantilever/calibration.json --output-dir runs/elasticity-calibration
```

校准检查 P2 制造解收敛、基准/加密网格位移差异、`1e-8` 代数残差、`1e-5` 力平衡、`1e-6` 载荷线性以及内存和耗时。任何硬门禁失败都应停止并诊断求解器或网格，不能静默删除失败样本。

重点检查 `runs/elasticity-calibration/diagnostics/calibration.json` 中每个样本的 `solve_seconds`、`observed_peak_rss_mb` 和 `mesh_relative_l2`，据此估算 Smoke 与 Full 的有限元总成本和磁盘规模。

## 5. Smoke 开发闭环

校准通过后运行：

```powershell
uv run surrogate-loop elasticity2d run --config examples/elasticity_2d_cantilever/smoke.json --runs-dir runs --request "训练二维悬臂梁位移场代理模型"
```

Smoke 使用开发测试集，可以指导模型和工程调整，因此只形成开发证据。报告应同时呈现 DeepONet 与 POD-RBF 指标、训练记录、代表性位移真值/预测/绝对误差图，以及来自 FEniCSx 的应力和 von Mises 诊断图。应力图只用于检查有限元标签和局部高梯度，不是网络预测结果。

检查 `development_evaluation.json`、阶段摘要、`datasets/dataset_manifest.json`、数据摘要、求解器版本、POD-RBF/DeepONet 的数据身份、训练集归一化，以及位移和应力诊断图的样本身份。

如果运行中断，只有身份和摘要完全一致的成功阶段可以复用。状态为 `failed` 的同一运行身份不得原地覆盖恢复；修改请求文字或配置会产生新的运行身份。

## 6. Full 与一次性验收

Full 不能由本指南中的命令链自动触发。只有校准和 Smoke 结果完成评审，并再次获得明确确认后，才可单独执行：

```powershell
uv run surrogate-loop elasticity2d run --config examples/elasticity_2d_cantilever/full.json --runs-dir runs --request "执行已确认的二维悬臂梁 Full 训练与一次性封存验收"
```

Full 固定使用 512/96/128 个训练、验证、封存测试样本。封存场在模型、阈值、归一化和推理代码冻结前不可读；每个运行身份只能消费一次。DeepONet 必须独立达到全场位移中位/P95/最差 `3%/8%/15%`、自由端和柔度门槛及至少 100 倍 CPU 加速。POD-RBF 只能作为对照，不能替代 DeepONet 验收。

## 7. 可信推理

只有状态为 `accepted`、文件摘要完整的 Full 产物能通过正常推理入口加载。点预测：

```powershell
uv run surrogate-loop elasticity2d predict --run-dir runs/<accepted-run-id> --e 3 --nu 0.3 --p 0.006 --theta -1.5707963268 --y0 0.5 --w 0.12 --x 4 --y 0.5
```

规则网格预测：

```powershell
uv run surrogate-loop elasticity2d predict --run-dir runs/<accepted-run-id> --e 3 --nu 0.3 --p 0.006 --theta -1.5707963268 --y0 0.5 --w 0.12 --nx 129 --ny 33 --output predicted_displacement.npz
```

参数或坐标越界、NaN/Inf、摘要损坏、未验收运行、试图覆盖受保护产物都会在加载权重或前向传播前被拒绝。当前接口不静默裁剪，也不保证训练域外外推。

## 8. 运行产物和常见故障

典型运行目录包含请求、解析后配置、样本计划、求解作业、开发与测试数据、求解器诊断、POD-RBF、模型权重、归一化统计、状态和验收摘要。`runs/` 不提交 Git。

- 找不到 Conda/Miniforge：确认 `conda.exe` 可用，或安装在默认 Miniforge 路径后重开终端。
- FEniCSx 即时编译失败：确认 Visual Studio Build Tools 的 MSVC 与 Windows SDK 已安装。
- `petsc4py_available=true`：环境不符合本仓库锁定的 Windows PyAMG 合同，应重建环境。
- 中文 Windows 下 Conda 输出乱码或编码异常：更新到包含 UTF-8 子进程修复的代码，不要改用 `shell=True`。
- 物理门禁失败：查看 `diagnostics/solver_quality.json` 或校准报告；不得删除失败样本继续训练。
- SHA-256 或协议失败：视为产物损坏或版本不匹配，不能绕过校验。
- CUDA 不可用：`device=auto` 会回退 CPU；显式 `device=cuda` 会失败。有限元求解始终在 CPU。
- 训练达到软预算：保留诊断并停止，不会静默缩小 Full 模型或放宽门槛。
