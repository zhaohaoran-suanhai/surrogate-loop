# 一维热传导闭环操作指南

## 1. 安装与验证

```powershell
uv sync --extra operator --all-groups
uv run python -c "import torch; print(torch.__version__, torch.version.cuda, torch.cuda.is_available())"
uv run ruff check .
uv run pytest
```

本机已验证环境为 Python 3.11.9、PyTorch 2.9.0+cu126、CUDA 12.6 和 NVIDIA GeForce RTX 4060 Laptop GPU。

## 2. 运行 Smoke

```powershell
uv run surrogate-loop operator validate --config examples/heat_1d_operator/smoke.json
uv run surrogate-loop operator run --config examples/heat_1d_operator/smoke.json --runs-dir runs --request "训练并验收一维热传导 DeepONet Smoke 闭环"
```

最终 JSON 返回 `run_dir`、`status`、`deeponet_metrics` 和 `pod_metrics`。状态含义：

- `accepted`：DeepONet 独立通过全部测试集门槛，可以加载推理。
- `rejected`：训练和评价完成，但至少一个 DeepONet 指标不达标；保留诊断，禁止正常推理。
- `failed`：配置、求解器、训练或产物阶段发生错误，查看 `error.json`。

## 3. 查看报告

```powershell
uv run surrogate-loop operator report --run-dir runs/运行标识
```

重点查看训练停止原因、最佳 epoch、求解器解析误差、DeepONet 测试指标和 POD/GPR 对照指标。

## 4. 推理

单点预测：

```powershell
uv run surrogate-loop operator predict --run-dir runs/运行标识 --alpha 0.1 --a 1.0 --b 0.1 --x 0.5 --t 0.25
```

完整规则网格场：

```powershell
uv run surrogate-loop operator predict --run-dir runs/运行标识 --alpha 0.1 --a 1.0 --b 0.1 --nx 129 --nt 101 --output predicted_field.npz
```

超出训练参数域、超出 `(x,t) ∈ [0,1]^2`、非有限输入、产物哈希损坏以及非 accepted 运行都会被拒绝。

## 5. 资源与故障处理

- 首次 CUDA wheel 下载约 2.4 GiB，本机首次准备约 28 分钟；这是一次性环境成本。
- Smoke 预算上限为 10 分钟；锁定配置实测约 97 秒完成整个 CLI，其中 GPU 训练约 86 秒。Full 软预算为 60 分钟。
- 训练时建议接通电源并使用高性能模式。
- PyTorch 缺失时运行 `uv sync --extra operator --all-groups`。
- `torch.cuda.is_available()` 为 False 时，`device: auto` 会退回 CPU；CPU Full 可能需要数小时。
- CUDA OOM 时降低 `query_batch_size`，不降低数据规模、网络结构或验收门槛。
- 求解器未通过解析解校验时禁止训练，应检查离散实现和网格阈值。
- DeepONet rejected 时先检查归一化、训练曲线和误差场，不事后降低验收标准。

## 6. 实测记录

### 6.1 Smoke

2026-07-16 使用锁定的 `smoke.json` 在 NVIDIA GeForce RTX 4060 Laptop GPU 上完成正式运行：

- 运行标识：`heat-20260716T121750Z-a24cfaa6`
- 最终状态：`accepted`
- CLI 墙钟时间：约 96.8 秒
- DeepONet 训练时间：86.36 秒
- 停止原因：达到 1500 epoch 上限；最佳 epoch 为 1494
- 峰值 CUDA 已分配显存：19.21 MiB
- DeepONet 中位/p95/最坏逐算例相对 L2：1.51% / 3.31% / 4.49%
- DeepONet 归一化 RMSE：3.08%
- DeepONet 初值相对 L2：`9.87e-8`
- DeepONet 边界最大绝对误差：`1.43e-7`
- POD/GPR 中位/p95/最坏逐算例相对 L2：1.39% / 4.95% / 6.14%

初始的 150 epoch 预算暴露出普通监督输出对初边值和整体场拟合不足。实现随后加入只编码已知初边值结构的硬约束层，并在不改变数据、网络宽度和验收门槛的前提下校准到 1500 epoch。正式 CLI 结果与校准结果一致，accepted 产物已通过 SHA-256 校验完成单点预测和 65×51 完整场导出。

### 6.2 Full

2026-07-16 使用锁定的 `full.json` 在同一 GPU 上完成正式运行：

- 运行标识：`heat-20260716T122109Z-561bc7bf`
- 最终状态：`accepted`
- CLI 墙钟时间：约 107.5 秒
- DeepONet 训练时间：88.13 秒
- 停止原因：达到 600 epoch 上限；最佳 epoch 为 578
- 峰值 CUDA 已分配显存：20.30 MiB
- 数值求解器 p95/最坏相对解析解误差：0.0024% / 0.0035%
- DeepONet 中位/p95/最坏逐算例相对 L2：0.731% / 1.462% / 3.013%
- DeepONet 归一化 RMSE：1.494%
- DeepONet 初值相对 L2：`8.76e-8`
- DeepONet 边界最大绝对误差：`1.09e-7`
- POD/GPR 中位/p95/最坏逐算例相对 L2：0.542% / 1.190% / 1.593%

Full 的五项 DeepONet 硬门槛全部通过，且 accepted 产物已通过哈希校验完成单点预测和 101×129 完整场导出。本问题具有低维解析模态结构，因此 POD/GPR 略优于 DeepONet 是合理的；它仍只作为诊断基线，不替代神经算子验收。本实验验证的是参数到完整时空场的可审计闭环，不代表对域外参数、任意 PDE 或真实 CFD 的泛化承诺。
