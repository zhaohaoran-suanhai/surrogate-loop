## Executive Summary

本仓库已经形成三个可审计的代理模型训练闭环：标量 ODE、一维热传导和二维线弹性。其中，二维线弹性首次把外部 FEniCSx 有限元求解、全新样本生成、物理质量门禁、三种子训练、模型冻结、一次性 sealed-test 和域内推理串成完整 Full 流程，运行 `elasticity-full-ba8ff8e584d9` 最终状态为 `accepted`。

本期最重要的业务结论是：在已冻结的二维悬臂梁、平面应力和参数域合同内，选中的 `directional_linear_v2` Vector DeepONet 在 128 个封存测试样本上实现全场相对 L2 中位误差 `0.2519%`、P95 `1.5152%`、最差 `4.4492%`；单点 CPU 推理相对 FEniCSx 中位求解达到 `931.18×` 加速。七项验收指标全部通过预先冻结的门槛。

这证明仓库已经具备“把一个明确、白名单化的物理问题做成可复现代理模型产品原型”的工程能力。它还不是任意 PDE 平台，也不构成生产认证、实验验证或域外泛化保证。下一阶段应优先巩固可演示性、状态治理和重复验收效率，而不是立即扩展第四个 PDE。

## 本周工作与项目目标

项目目标不是单独训练一个神经网络，而是建立自然语言驱动、结构化约束、确定性验证、全流程可审计的本机代理模型训练闭环。本周围绕第三个闭环完成了四件关键工作：

1. 用 FEniCSx/PyAMG 生成 736 个全新二维线弹性样本，并在进入训练前执行残差、力平衡、载荷线性和网格收敛门禁。
2. 用 512/96/128 的训练、验证、封存测试划分训练三种随机种子，以验证集损失选择 seed `20260718`，冻结后才一次性读取 sealed-test。
3. 完成全场误差、自由端、柔度、固支边界和 CPU 加速的七项确认性验收，运行状态进入 `accepted`。
4. 把三个闭环的结果组织成可供管理层快速理解、技术人员可以沿证据链复核的单一 Markdown 汇报。

## 基本架构：从受控需求到可信推理

仓库把“需求、数据、训练、验收、推理”设计为有边界的状态机，而不是一条可以任意跳步的脚本。

```mermaid
flowchart LR
    A["自然语言需求与人工确认"] --> B["白名单配置"]
    B --> C["数值求解与数据门禁"]
    C --> D["代理模型训练与验证选模"]
    D --> E["模型冻结与分级验收"]
    E --> F["摘要保护的域内推理"]
```

这套架构有三层约束：

- 输入约束：配置只接受已实现的方程模板、参数范围、预算和验收门槛，拒绝任意方程执行。
- 数据约束：样本身份、数据划分和质量诊断写入运行产物；归一化器只能在训练集上拟合。
- 证据约束：calibration、Smoke 与 Full 含义不同；Full 先冻结开发产物，再一次性消费 sealed-test，只有验收摘要完整且门槛全部通过才进入 `accepted`。

当前稳定入口见[项目文档地图](../README.md)、[当前能力与状态](../当前能力与状态.md)和[二维线弹性操作指南](../guides/二维线弹性闭环操作指南.md)。

## 核心算法：把物理结构写进数据和模型

### 标量闭环：验证集选模

强迫反应问题在固定数据划分上比较 PRS、GPR 与 MLP。候选模型只用训练集拟合，验证集负责选模，测试集只对最终冻结模型评估一次。这一闭环建立了后续 PDE 模型共同遵守的数据纪律。

### 一维热传导：数值解、解析解与算子学习

一维热方程采用 Crank–Nicolson 生成数值轨迹，并用可用的解析解进行正确性核查。POD/GPR 提供低维基线，DeepONet 用 Branch 网络编码问题参数或输入条件、用 Trunk 网络编码查询坐标 `(x,t)`，学习从问题实例到温度场的算子映射。

### 二维线弹性：有限元真值与结构化 Vector DeepONet

二维悬臂梁采用 FEniCSx 离散平面应力线弹性方程，并由 PyAMG 支撑大规模线性系统求解。样本进入训练前必须通过相对残差、合力平衡、载荷线性和网格收敛诊断，避免把数值求解缺陷当成学习目标。

`directional_linear_v2` 不让网络重复学习线弹性中已知的线性结构：

- Branch 输入为 `(nu, y0, w)`，表示泊松比、载荷中心与载荷宽度；Trunk 输入为查询坐标 `(x,y)`。
- 网络输出四个方向响应通道，分别组合为二维位移的基响应。
- 任意载荷角通过 `cos(theta)` 与 `sin(theta)` 对方向响应做解析叠加。
- 位移幅值按 `P/E` 缩放，把载荷大小和杨氏模量的已知线性关系写进输出。
- 输出乘以固支边界乘子，使 `x=0` 的位移严格为零，而不是仅依赖损失函数近似学习。

训练阶段的归一化统计只来自 512 个训练样本。三个种子在 96 个验证样本上选模，最终 seed `20260718` 的最佳 epoch 为 `580`，验证损失为 `5.713795326300897e-05`。冻结开发产物后，128 个 sealed-test 样本只消费一次并生成不可混用的摘要与哈希证据。

## 二维线弹性成为首个外部有限元 Full 闭环

二维问题是本期汇报主线，因为它同时验证了科学计算环境、机器学习环境和审计协议之间的边界。

问题合同固定为长度 `4.0`、高度 `1.0` 的二维悬臂梁和平面应力假设。参数域包括 `E∈[1,5]`、`nu∈[0.2,0.45]`、载荷 `P∈[0.002,0.01]`、方向 `theta∈[-π,π]`、载荷中心 `y0∈[0.2,0.8]` 和宽度 `w∈[0.08,0.2]`。代理模型输出域内任意查询点的二维位移，而不是仅输出一个标量指标。

完整运行使用 736 个全新 FEniCSx 样本：512 个训练、96 个验证、128 个封存测试。FEniCSx 数据生成约占总耗时的主体；三种子训练耗时分别约 `47.21`、`43.35` 和 `45.73` 秒，总训练约 `2 分 16 秒`。成功 Full 的总墙钟约 `1 小时 16 分 38 秒`，说明当前成本瓶颈在高保真数据求解，而不是神经网络训练。

首次运行 `elasticity-full-a12ace8d157e` 在正式样本生成前因 FFCx 用户缓存权限失败而停止。该失败保留为环境审计记录，没有生成正式训练样本、没有开始训练，也没有消费 sealed-test。恢复缓存权限后，以新身份 `elasticity-full-ba8ff8e584d9` 从受控请求重新执行并完成验收。

## Full 封存验收全部通过

| 指标 | 实测 | 门槛 | 结论 |
|---|---:|---:|---|
| 全场相对 L2 中位 | 0.2519% | ≤ 3% | 通过 |
| 全场相对 L2 P95 | 1.5152% | ≤ 8% | 通过 |
| 全场最差误差 | 4.4492% | ≤ 15% | 通过 |
| 自由端误差 P95 | 0.7550% | ≤ 8% | 通过 |
| 柔度误差 P95 | 3.2833% | ≤ 10% | 通过 |
| 固支边界误差 | 0 | ≤ `1e-7` | 通过 |
| CPU 加速 | 931.18× | ≥ 100× | 通过 |

基准中 FEniCSx 单样本中位耗时为 `4.60537 s`，神经网络中位推理耗时为 `0.004946 s`。这里的 `931.18×` 是当前机器、当前计时方法和当前查询合同下的 CPU 对比，不应外推为所有硬件、网格规模或批处理负载下的固定加速倍数。

POD/RBF 基线在同一 sealed-test 上的全场相对 L2 中位为约 `1.944%`，但 P95 和最差分别达到约 `35.923%` 与 `94.074%`，没有满足完整验收合同。Vector DeepONet 的优势因此不仅体现在平均精度，也体现在尾部风险控制。

## 当前仓库的能力边界

### 已经具备

- 在三个已实现模板内，将白名单配置转为可审计的采样、训练、冻结、验收和推理产物。
- 区分 calibration、Smoke 和 Full 证据等级，并对 Full 执行一次性 sealed-test。
- 对二维悬臂梁完成真实 FEniCSx 数据生成、物理门禁、三种子验证选模和 Full `accepted`。
- 在 accepted 运行、训练参数域和完整摘要存在的前提下执行受保护的域内点预测。
- 通过 JSON 产物、SHA-256 身份、状态文件和报告命令追溯一次运行的关键决策。

### 尚不具备

- 不是“输入任意方程即可自动训练”的任意 PDE 平台；新增 PDE 仍需显式设计和实现。
- 不支持任意几何、任意边界条件、三维弹性、大变形、材料非线性、塑性、接触或动力学。
- 当前 Full 只覆盖已冻结参数域，不提供域外误差或外推安全保证。
- 结果尚未与实物实验、商业求解器或独立第三方数据做系统验证。
- 没有形成多用户服务、权限、监控、资源调度、模型注册和回滚等生产部署能力。
- 哈希能检查本地证据一致性，但目前没有带外数字签名、时间戳服务或不可篡改外部存证。
- 主要验收对象是位移、自由端、柔度与边界误差；没有把网络应力/应变场精度纳入本次 Full 验收。

因此，“已完成闭环”应理解为：针对已实现问题模板和冻结合同，工程流程与域内确认性精度已走通；不等价于通用仿真替代平台或生产级数字孪生。

## 下一阶段建议与待确认问题

建议下一阶段仍不增加新 PDE，优先完成以下工作：

5. 决定产品化方向：是继续做本机研发工作台，还是投入模型注册、服务化、权限和监控能力。

## Caveats and Assumptions

- 本报告中的 accepted 表示预先冻结合同下的域内确认性验收，不代表生产认证、实验验证或域外泛化保证。
- 误差统计基于 128 个一次性 sealed-test 样本；它刻画当前采样合同，不是所有可能载荷和材料组合的数学上界。
- 加速比依赖当前硬件、FEniCSx 配置、神经网络计时和单点查询方式；跨机器或批量场景需重新基准。
- Full 数据来自数值求解器，不替代实验数据；有限元离散、材料假设和边界条件本身仍是模型误差来源。
- 本报告不修改历史设计和实施计划；历史文档应按其记录时点理解，当前状态以稳定状态页和 accepted 运行产物为准。
