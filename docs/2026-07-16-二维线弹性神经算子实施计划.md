# 二维线弹性神经算子实施计划

> **面向代理执行者：** 必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans`，严格按任务顺序执行；每个步骤使用复选框跟踪。

**目标：** 在现有仓库中建立 Windows 原生 FEniCSx 0.11 数据生成、POD-RBF 基线、Vector DeepONet 训练、封存测试验收和可信推理组成的二维线弹性最小闭环。

**架构：** 当前 uv/PyTorch 环境负责结构化配置、采样、训练、验收与推理；独立 Conda/FEniCSx 环境负责有限元求解、插值和物理门禁。两端只通过版本化 JSON/NPZ 协议通信，并把可复用的外部求解器适配、场数据、Vector DeepONet 和场评价能力沉淀到 `surrogate_loop.operator`。

**技术栈：** Python 3.11、uv、Pydantic 2、NumPy、SciPy、scikit-learn、PyTorch 2.9、Windows Miniforge/Conda、FEniCSx/DOLFINx 0.11、PyAMG、pytest、Ruff。

## 全局约束

- 直接在 `main` 分支实施，不创建额外分支或 worktree。
- 用户可读规格、计划、指南和算例说明全部使用中文，并直接放在 `docs/` 既有层级中，不创建 `docs/superpowers/`。
- 当前 uv 环境保持 Python `>=3.11,<3.12`；FEniCSx 使用独立 Conda 环境 `surrogate-loop-fenicsx-0.11`。
- Windows FEniCSx 0.11 原生包使用 PyAMG/SciPy 串行求解，不假定 PETSc/petsc4py 可用。
- Full 参数域、`129×33` 观测网格、`512/96/128` 划分和 `3%/8%/15%` 位移误差门槛不得事后放宽。
- Full 封存测试场必须与开发数据物理分文件保存；训练和选模代码不得获得封存文件内容。
- 所有归一化统计量只能由训练集拟合；测试集只在模型、阈值和推理代码冻结后评价一次。
- 禁止 `eval`、`exec`、任意方程执行、配置驱动的动态导入和配置驱动的任意系统命令。
- 每项功能先提交失败测试，再写最小实现；每个任务结束运行目标测试、完整普通测试和 Ruff。
- 现有标量闭环和一维热传导闭环必须保持兼容。
- `runs/` 只保存本地产物，不提交训练数据、模型或运行目录。
- Full 数据生成与训练不在本计划的自动执行范围内；Smoke 通过后必须再次取得用户明确确认。

```python
def build_solver_command(action: str, *arguments: str) -> list[str]:
    if action not in ALLOWED_ACTIONS:
```

```python
@classmethod
def fit(cls, features, coordinates, fields):
    return cls(
        feature_mean=features.mean(axis=0),
        feature_std=_safe_std(features, axis=0),
        coordinate_mean=coordinates.mean(axis=0),
        coordinate_std=_safe_std(coordinates, axis=0),
        target_rms=np.maximum(np.sqrt(np.mean(fields**2, axis=(0, 1))), 1e-12),
    )
```

目标只保存每分量 RMS，不保存目标均值；这保证零位移边界的物理语义不被偏移。
