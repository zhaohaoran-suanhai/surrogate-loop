| 闭环 | 输入 | 输出 | 数据来源 | 代理模型 | 当前最高证据 | 可可信推理 |

## 证据等级

| 等级 | 能证明什么 | 不能证明什么 |
|---|---|---|
| 假求解 E2E | CLI 编排、保护机制、产物兼容性 | 外部求解器设置、物理门禁和科学正确性 |
| 真实外部求解器 E2E | 微型跨环境协议和真实 FEniCSx/Fluent 链路 | 正式数据规模上的精度与稳定性 |
| calibration | 数值方法、网格、物理门禁和本机资源基线 | 神经算子精度 |
| Smoke | 完整开发流程、模型诊断和开发测试表现 | 未参与调参的确认性结论 |
| Full | 冻结合同下一次性封存测试的确认性验收 | 域外泛化、生产认证或任意物理问题 |

Smoke 可能产生 `accepted` 或 `development_complete` 等开发状态，但不能替代 Full 的一次性确认性语义。二维线弹性和二维方腔的最高证据均为上述 `accepted` Full，各自的 Smoke 仍只用于追溯开发过程。

## 功能边界

- 运行时 CLI 不调用 LLM；自然语言由协作 Agent 映射到人工确认、受审查的白名单配置。
- 不解析或执行任意方程、任意 Python、动态插件或配置指定代码。
- 不支持任意 PDE、任意几何、三维或非线性弹性，也不自动承诺训练域外外推。
- 当前方腔模型只覆盖固定单位方腔、固定顶盖速度、稳态不可压层流、固定网格和 `Re∈[10,400]`；不支持瞬态、湍流或其他边界条件。
- 当前二维线弹性网络只正式预测位移；应力、应变和 von Mises 图是 FEniCSx 标签诊断。
- 不默认启动新的 Full，不因 Smoke 指标不理想而事后放宽 Full 门槛；读取已有 accepted 报告与域内推理不等于授权重新训练或消费新的 sealed-test。
- `runs/`、训练数据、模型和本地报告不提交 Git；运行身份和摘要必须匹配才能复用阶段。
