# 当前能力与状态

## 一页结论

仓库已有三个可运行闭环：标量 ODE、一维热传导和二维线弹性。它们都遵循“自然语言意图 → 受约束配置 → 数值数据 → 代理模型训练 → 评价与产物 → 受保护推理”的总体路线，但证据等级不同。

- 标量 ODE 已形成可验收、可重载推理的最小闭环。
- 一维热传导已有状态为 `accepted` 的 Full 确认性运行，可通过正常入口执行可信域内推理。
- 二维线弹性已完成 calibration 和状态为 `development_complete` 的 Smoke，流程已经走通，但二维线弹性当前未完成 Full，不能通过正常可信推理入口加载为已验收模型。

## 能力矩阵

| 闭环 | 输入 | 输出 | 数据来源 | 代理模型 | 当前最高证据 | 可可信推理 |
|---|---|---|---|---|---|---|
| 标量 ODE | `gamma` | `u(1)` | 固定 ODE 数值求解 | PRS、GPR、MLP 候选选模 | 已验收最小闭环 | 是，限训练域和清单完整产物 |
| 一维热传导 | `(alpha,A,B)` 与 `(x,t)` | `u(x,t)` | Crank–Nicolson，解析解校验 | DeepONet，POD/GPR 诊断基线 | `accepted` Full | 是，限 accepted Full 与训练域 |
| 二维线弹性 | `(E,nu,P,theta,y0,w)` 与 `(x,y)` | `(u_x,u_y)` | 独立 FEniCSx 0.11 / PyAMG | `directional_linear_v2` Vector DeepONet，POD-RBF 诊断基线 | `development_complete` Smoke | 否，尚无 accepted Full |

“可可信推理”只表示仓库的正常推理入口会加载通过相应验收且摘要完整的域内模型，不表示生产认证、训练域外泛化或任意 PDE 支持。

## 证据等级

| 等级 | 能证明什么 | 不能证明什么 |
|---|---|---|
| 假求解 E2E | CLI 编排、保护机制、产物兼容性 | FEniCSx 方程、物理门禁和科学正确性 |
| 真实 FEniCSx E2E | 微型跨环境协议和真实求解器链路 | 正式数据规模上的精度与稳定性 |
| calibration | 数值方法、网格、物理门禁和本机资源基线 | 神经算子精度 |
| Smoke | 完整开发流程、模型诊断和开发测试表现 | 未参与调参的确认性结论 |
| Full | 冻结合同下一次性封存测试的确认性验收 | 域外泛化、生产认证或任意物理问题 |

Smoke 可能产生 `accepted` 或 `development_complete` 等开发状态，但不能替代 Full 的一次性确认性语义。二维线弹性当前只到 Smoke。

## 功能边界

- 运行时 CLI 不调用 LLM；自然语言由协作 Agent 映射到人工确认、受审查的白名单配置。
- 不解析或执行任意方程、任意 Python、动态插件或配置指定代码。
- 不支持任意 PDE、任意几何、三维或非线性弹性，也不自动承诺训练域外外推。
- 当前二维线弹性网络只正式预测位移；应力、应变和 von Mises 图是 FEniCSx 标签诊断。
- 不默认启动 Full，不因 Smoke 指标不理想而事后放宽 Full 门槛。
- `runs/`、训练数据、模型和本地报告不提交 Git；运行身份和摘要必须匹配才能复用阶段。
