# 仓库骨架与基础环境实施计划

> **供 Agent 工作者使用：** 必须使用 `subagent-driven-development`（推荐）或 `executing-plans`，逐项执行本计划。所有执行步骤使用复选框（`- [ ]`）跟踪。

**目标：** 建立可安装、可测试、目录职责清晰的 Python 3.11 仓库骨架，并用中文 README 提供统一入口。

**架构：** 本计划只建立后续闭环依赖的工程基础，不实现 ODE、数据生成或模型训练。核心包固定在 `src/surrogate_loop/`，算例固定在 `examples/`，测试按 `unit/integration/e2e` 分层，设计、计划和指南直接存放在 `docs/`。

**技术栈：** Python 3.11、uv、Pydantic 2、NumPy、SciPy、scikit-learn、Matplotlib、joblib、pytest、pytest-cov、Ruff、Hatchling。

## 全局约束

- 目标平台为 Windows，Python 版本必须为 `>=3.11,<3.12`。
- 虚拟环境固定为仓库内 `.venv`，依赖由 uv 管理，`uv.lock` 必须提交。
- README、设计、计划、指南、算例说明和 Agent 规则使用中文；Python 标识符、JSON 字段和第三方库名称保留英文。
- `docs/`、`src/surrogate_loop/`、`examples/`、`tests/`、`runs/` 的职责不得混用。
- 本计划不得实现任意方程解析、数据生成、模型训练、外部 LLM API、Web UI、PyTorch 或 GPU 支持。
- README 中尚未完成的 CLI 必须明确标记为“目标接口，尚未实现”。
- 不允许使用 `eval`、`exec` 或执行自然语言生成的 Python 代码。

from __future__ import annotations

import argparse
from collections.abc import Sequence

from surrogate_loop import __version__


def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(
        prog="surrogate-loop",
        description="标量代理模型最小闭环命令行入口",
    )
    parser.add_argument(
        "--version",
        action="version",
        version=f"%(prog)s {__version__}",
    )
    return parser


def main(argv: Sequence[str] | None = None) -> int:
    build_parser().parse_args(argv)
    return 0

import subprocess
import sys


def run_cli(*args: str) -> subprocess.CompletedProcess[str]:
    environment = os.environ.copy()
    environment["PYTHONIOENCODING"] = "utf-8"
    return subprocess.run(
        [sys.executable, "-m", "surrogate_loop", *args],
        check=False,
        capture_output=True,
        encoding="utf-8",
        env=environment,
    )


def test_module_entrypoint_displays_help() -> None:
    result = run_cli("--help")

    assert result.returncode == 0
    assert "标量代理模型最小闭环命令行入口" in result.stdout


def test_module_entrypoint_displays_version() -> None:

    assert result.returncode == 0
