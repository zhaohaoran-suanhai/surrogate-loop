# 二维线弹性神经算子闭环设计

## 1. 建设目标

在已经完成的标量代理模型闭环和一维热传导神经算子闭环之上，建立第一个由外部有限元求解器提供训练数据的二维场代理模型闭环。

本阶段固定使用 FEniCSx 作为二维线弹性训练数据生成器，PyTorch 继续负责神经算子训练，最终形成：

```text
自然语言问题
→ 结构化方程配置
→ FEniCSx 数值求解并生成数据
→ 数据质量门禁
→ 训练神经算子
→ 封存测试验收
→ 快速神经网络推理
```

第一版问题为固定矩形悬臂梁在参数化局部边界载荷下的平面应力响应，学习合同为：

\[
\mathcal G:(E,\nu,P,\theta,y_0,w)
\longmapsto
\mathbf u(\cdot,\cdot)=(u_x,u_y).
\]

首要目标不是构建任意有限元平台，而是验证一条可信、可复现、可审计、可扩展到后续二维 PDE 的最小闭环。

## 2. 已选技术路线

采用“FEniCSx 三角形细网格求解、插值到固定规则观测网格、Vector DeepONet 学习连续坐标场”的路线。

选择该路线的原因如下：

- FEniCSx 使用 Python/UFL 表达弱形式，适合与当前 Python 仓库衔接。
- 有限元网格与神经网络观测坐标解耦，求解网格可以独立加密，不把模型绑定到自由度编号。
- DeepONet 可以在标准观测网格上训练，同时保留对区域内任意坐标查询的能力。
- 现有一维热传导 DeepONet 的训练、产物和验收经验可以复用。
- 以后切换到 MFEM 时，只需实现相同的数据协议，不必重写训练侧。

以下方案不作为第一版主路线：

- 直接学习有限元自由度：会使模型与网格、单元阶次和自由度编号强耦合。
- FNO：适合规则网格批量整场预测，但当前数据原生来自三角形有限元网格，且与现有闭环差异较大。
- 直接预测应力：应力对位移梯度、网格和局部载荷更敏感，第一版先建立可信位移场闭环。

## 4. 数学问题

### 4.1 控制方程

区域固定为：

\[
\Omega=[0,L]\times[0,H],\qquad L=4,\quad H=1.
\]

在无体力条件下求解：

\[
-\nabla\cdot\boldsymbol\sigma(\mathbf u)=\mathbf 0.
\]

小应变张量为：

\[
\boldsymbol\varepsilon(\mathbf u)
=\frac12\left(\nabla\mathbf u+\nabla\mathbf u^{\mathsf T}\right).
\]

平面应力本构关系写为：

\[
\boldsymbol\sigma
=2\mu\boldsymbol\varepsilon
+\lambda_{\rm ps}\operatorname{tr}(\boldsymbol\varepsilon)\mathbf I,
\]

\[
\mu=\frac{E}{2(1+\nu)},
\qquad
\lambda_{\rm ps}=\frac{E\nu}{1-\nu^2}.
\]

### 4.2 边界条件

- 左端 \(x=0\)：完全固支，\(\mathbf u=\mathbf 0\)。
- 上、下边界：零牵引。
- 右端 \(x=L\)：施加平滑局部牵引。

右端牵引定义为：

\[
\mathbf t(y)=P\,g(y;y_0,w)
\begin{bmatrix}
\cos\theta\\
\sin\theta
\end{bmatrix},
\]

其中截断高斯形状函数在右边界上归一化：

\[
g(y;y_0,w)=
\frac{\exp\left[-\frac12\left(\frac{y-y_0}{w}\right)^2\right]}
{\int_0^H\exp\left[-\frac12\left(\frac{s-y_0}{w}\right)^2\right]\,ds},
\]

因此 \(P\) 始终表示右边界总合力的大小，便于跨样本比较和力平衡验证。

### 4.3 参数域

\[
E\in[1,5],\qquad
\nu\in[0.20,0.45],\qquad
P\in[0.002,0.01],
\]

\[
\theta\in[-\pi,\pi],\qquad
y_0\in[0.2,0.8],\qquad
w\in[0.08,0.20].
\]

该无量纲参数域满足 \(P/E\le 10^{-2}\)，用于控制小变形响应尺度。第一版先验证无量纲闭环，后续再增加参考量纲到工程单位的显式映射。

## 7. FEniCSx 数值设计

### 7.1 离散与求解

- 使用矩形分区后剖分得到的三角形网格。
- 使用二次 \(P_2\) 向量 Lagrange 有限元。
- Full 候选基准网格为 \(256\times64\) 个矩形分区再三角剖分。
- 使用 DOLFINx 原生稀疏矩阵接口组装线性系统。
- Windows 原生环境以 PyAMG 迭代求解为主，SciPy 稀疏直接法作为校准和故障诊断候选。
- 校准阶段从固定白名单中比较直接法和迭代法，随后冻结 Full 求解器设置。
- 训练标签以 `float64` 生成和保存，进入 PyTorch 后转换为 `float32`。

### 7.2 观测网格

有限元位移插值到固定规则观测网格：

\[
N_x\times N_y=129\times33,
\]

输出张量为：

```text
[33, 129, 2]
```

观测网格包含左右、上下边界。有限元求解网格可以独立加密，模型数据身份由观测坐标、插值规则和求解网格清单共同确定。

第一版只把 \((u_x,u_y)\) 保存为正式训练标签。应力、应变和 von Mises 用于：

- 求解器验证；
- 代表性真值图；
- 局部高梯度诊断；
- 后续应力代理模型设计的参考。

### 7.3 求解器强制质量门禁

在生成正式训练数据前必须通过：

1. **制造解测试**：独立验证弱形式、本构、边界施加和误差收敛。
2. **网格收敛**：校准样本在基准网格与加密网格之间的位移相对 \(L^2\) 差异不超过 `1%`。
3. **线性残差**：归一化代数残差不超过 `1e-8`。
4. **固支边界**：左端有限元位移自由度在数值精度内为零。
5. **力平衡**：左端总反力与右端总载荷的相对误差不超过 `1e-5`。
6. **载荷线性抽查**：固定其他参数时，载荷加倍后的位移与两倍原位移相对差异不超过 `1e-6`。
7. **基本完整性**：参数、位移、诊断量和插值结果全部有限且形状符合协议。

任一硬门槛失败时，整批数据生成任务失败并输出诊断报告。不能通过静默删除失败样本继续训练，以免引入选择偏差。

## 8. 参数采样与数据划分

采用固定种子的分层 Latin Hypercube 采样。角度 \(\theta\) 在物理配置中保持可读表示，进入模型前转换为：

\[
(\cos\theta,\sin\theta),
\]

避免 \(-\pi\) 与 \(\pi\) 处的人为不连续。

训练、验证和测试参数在有限元求解前即确定。数据分为三级：

| 阶段 | 用途 | 样本规模 |
|---|---|---:|
| 校准 | 单次求解耗时、内存、PyAMG/SciPy 求解器和网格误差 | 16 |
| Smoke | 闭环调试与模型开发 | 96 训练 / 24 验证 / 24 开发测试 |
| Full | 正式训练和确认性验收 | 512 训练 / 96 全新验证 / 128 封存测试 |

阶段复用规则：

- 校准样本用于求解器和网格选择，不承担最终模型验收。
- Smoke 的非封存数据可以在 Full 阶段纳入训练开发池，避免浪费有限元结果。
- Full 验证集和封存测试集必须重新、独立采样。
- Smoke 开发测试结果可以指导工程设计，因此不能成为 Full 封存测试的一部分。
- Full 的 128 个测试样本在模型、归一化、超参数、阈值和推理代码冻结前不可读取结果。
- 同一 Full 运行只允许完成一次确认性测试；测试后调参必须创建新的运行身份和新的封存测试集。

Full 不会在环境安装后立即启动。必须先完成 16 个校准样本，记录每次求解耗时、峰值内存和磁盘规模，再决定是否执行 Smoke 和 Full。成本不可接受时，应调整网格或求解器并重新通过网格收敛门禁，而不是降低最终测试独立性。

## 9. 版本化数据协议

两个环境使用带 `schema_version` 和 `problem_id` 的协议。

### 9.1 求解任务 JSON

至少包含：

- 协议版本和问题标识；
- 参数样本及样本身份；
- 区域、网格和有限元阶次；
- 观测坐标定义；
- 求解器白名单选项；
- 质量门槛；
- 随机种子和输出身份。

### 9.2 数据集 NPZ

统一场数据契约为：

```text
parameters   [n_samples, n_parameters]
coordinates  [n_points, n_spatial_dimensions]
fields       [n_samples, n_points, n_components]
split        [n_samples]
diagnostics  求解器和物理校验所需数组
```

Full 数据不放入一个可被训练进程整体加载的 NPZ。至少物理分离为：

```text
datasets/development.npz
datasets/sealed_test.npz
```

训练和选模阶段只能获得 `development.npz` 的路径与摘要。`sealed_test.npz` 由验收状态机在冻结模型后单独加载；其身份可以提前记录，但场数组不能提前进入训练进程。Smoke 数据使用同一协议，但明确标记为开发证据而非确认性证据。

二维弹性实例中：

```text
n_parameters = 6
n_spatial_dimensions = 2
n_components = 2
```

物理参数仍保存原始 \(\theta\)，模型特征转换另行写入训练配置和产物清单。

### 9.3 数据清单 JSON

至少记录：

- FEniCSx、DOLFINx、UFL、PyAMG、SciPy、MPI、Python 和 NumPy 版本；
- 网格规模、单元、有限元阶次和求解器设置；
- 参数域、采样种子和数据划分成员；
- 观测网格和插值规则；
- 各项求解器诊断统计；
- 数组形状、数据类型和 SHA-256 摘要；
- 生成状态、失败原因和时间成本。

协议版本不支持、文件缺失、摘要不符或诊断失败时，训练侧必须拒绝数据集。

## 10. 模型设计

### 10.1 POD-RBF 诊断基线

基线流程为：

1. 将每个训练位移场展平为统一向量。
2. 仅在训练集拟合 POD。
3. 按预先配置的累计能量比例保留模态，并设置模态数上限。
4. 使用 RBF 从结构化参数特征映射到 POD 系数。
5. 在与 DeepONet 完全相同的验证集和测试集上重构并评价。

POD-RBF 用于诊断数据维数、可学习性和 DeepONet 训练链路。第一版必须报告对比结果，但不强制 DeepONet 一定优于 POD-RBF。

### 10.2 Vector DeepONet

Branch 和 Trunk 分别编码参数及坐标：

\[
\mathbf z_b=B(\mathbf p),
\qquad
\mathbf z_t=T(x/L,y/H).
\]

两个位移分量共享 Branch 和 Trunk，仅输出头独立：

\[
u_{{\rm raw},c}
=\mathbf w_c^{\mathsf T}(\mathbf z_b\odot\mathbf z_t)+b_c,
\qquad c\in\{x,y\}.
\]

当前问题严格利用线弹性尺度关系和固支边界：

\[
\widehat{\mathbf u}(x,y)
=\frac{P}{E}\frac{x}{L}
\mathbf u_{\rm raw}
\left(\nu,\cos\theta,\sin\theta,y_0,w;x/L,y/H\right).
\]

因此：

- \(x=0\) 的固支条件在浮点精度内严格满足；
- \(\mathbf u\propto P/E\) 的线弹性尺度关系严格满足；
- 网络只学习与 \(\nu,\theta,y_0,w\) 有关的非平凡空间形状变化。

通用 Vector DeepONet 不硬编码该关系，而是支持由已注册问题提供受控的输入和输出变换。推理 API 仍要求完整六个物理参数。

### 10.3 训练策略

- 参数缩放和位移尺度只能从训练集拟合。
- 位移分量使用训练集 RMS 尺度归一化，不通过全场均值偏移破坏零边界语义。
- 损失为两个分量的尺度归一化 MSE，并按参数样本均衡。
- 使用“参数样本批次 × 坐标子批次”训练，避免一次展开所有组合。
- 验证和测试始终在完整 \(129\times33\) 观测网格计算。
- Smoke 用于冻结网络宽度、潜空间维数、学习率、调度和早停规则。
- Full 使用冻结配置运行 3 个固定随机种子，仅依据验证集选择最终模型。
- 模型只保存受支持的权重、结构配置、训练集归一化统计和数据身份，不保存不可审计的任意 Python 对象。

## 11. 推理能力

推理支持：

1. 返回标准 \(129\times33\) 位移场，用于批量评价和可视化。
2. 接受区域内任意查询坐标，体现 DeepONet 连续坐标表示能力。

推理前必须检查：

- 六个参数均为有限数；
- 参数全部位于已验收训练域；
- 坐标位于固定区域内；
- 运行状态为 `accepted`；
- 模型、配置、归一化器和数据身份相互匹配；
- 产物清单与摘要完整。

越界请求必须明确拒绝，不能静默裁剪或外推。

可以从网络位移对坐标的自动微分生成应变和应力诊断图，但第一版不对其工程精度作承诺，也不把它们纳入正式通过标准。

```powershell
uv run surrogate-loop elasticity2d calibrate --config <配置>
uv run surrogate-loop elasticity2d run --config <配置>
uv run surrogate-loop elasticity2d predict --run-dir <目录> --input <参数>
```

第一版的自然语言入口由 Codex 或其他受控上层工具把用户已确认的需求转换为白名单 JSON。仓库运行时不直接解析任意自然语言或任意方程。

`run` 的固定阶段为：

1. 验证配置、参数域和双环境。
2. 固化参数样本和数据划分。
3. 通过固定适配器调用 FEniCSx 环境。
4. 验证数据协议、摘要和全部物理门禁。
5. 训练并评价 POD-RBF。
6. 训练 Vector DeepONet 并依据验证集选择模型。
7. 冻结模型、归一化器、阈值和推理实现。
8. Full 模式下打开一次封存测试集。
9. 输出中文验收报告、误差图和推理示例。

```text

├── samples.json
├── datasets/
│   ├── development.npz

│   └── dataset_manifest.json

├── baseline/
├── model/
├── acceptance/

```

所有阶段采用原子写入。中断恢复时，只能复用配置身份和摘要完全一致的成功阶段。生成数据、模型和图表保存在 `runs/`，不提交 Git。

## 13. 代理模型验收

Full 的 128 个封存测试样本必须同时满足：

### 13.1 位移场

- 逐样本全场位移相对 \(L^2\) 误差中位数不超过 `3%`。
- 逐样本全场位移相对 \(L^2\) 误差 P95 不超过 `8%`。
- 最差样本全场位移相对 \(L^2\) 误差不超过 `15%`。
- 所有预测值有限。

### 13.2 工程量

- 右端平均位移向量的归一化误差 P95 不超过 `8%`。
- 柔度相对误差 P95 不超过 `10%`。
- 固支边界误差达到所用浮点精度。

### 13.3 推理速度

- 同一台机器、同一 CPU 条件下，单样本神经网络推理相对单样本 FEniCSx 组装、求解和插值至少加速 `100` 倍。
- 神经网络基准使用预热后的多次推理中位数。
- FEniCSx 基准不计 Conda 进程启动时间，但计入该参数样本的有限元组装、线性求解和观测点插值。
- 硬件、线程数、软件版本和测量次数必须进入报告。

POD-RBF 使用相同指标进入报告，但不能代替 DeepONet 的绝对验收。DeepONet 未通过任一硬门槛时，运行状态为 `rejected`，保留全部诊断产物但不能通过正常推理入口加载。

上述模型、阈值和 Full 测试身份必须在读取封存结果前锁定，不能根据 Full 测试结果事后降低标准。

## 14. 失败处理

- 配置无效：在采样、求解和训练前拒绝。
- Conda 或 FEniCSx 环境缺失：`doctor` 给出准确诊断，不创建伪数据。
- 求解器版本不匹配：拒绝执行正式数据生成。
- FEniCSx 子进程非零退出、超时或部分输出：标记 `failed`，禁止训练。
- 制造解、网格收敛、残差、固支、力平衡或线性检查失败：标记 `failed`，禁止训练。
- 数据文件缺失、协议版本不支持或摘要不匹配：拒绝加载。
- 训练出现 NaN/Inf 或设备错误：保存诊断和最后正常状态，标记 `failed`。
- 达到软资源预算：保存阶段状态并停止，不静默缩小模型或降低验收标准。
- Full 封存测试已读取后请求继续调参：要求创建新运行，不能覆盖原验收。
- 模型、配置、归一化器或数据身份不一致：拒绝推理。
- 参数或坐标越界：在加载模型和执行前向传播前拒绝。

## 15. 测试策略

实现遵循测试驱动开发：先写失败测试，再写满足规格的最小实现。

### 15.1 普通 uv 环境单元测试

这些测试不要求安装 FEniCSx：

- 配置字段、范围、未知字段和阶段约束。
- 角度周期特征和物理尺度变换。
- Latin Hypercube 采样、固定种子和数据划分隔离。
- JSON/NPZ 数据契约、协议版本、数组形状和摘要验证。
- 多维坐标、多分量场数据集。
- Vector DeepONet 输出形状、梯度、硬边界和 \(P/E\) 尺度关系。
- POD-RBF 的拟合、保存、重载和评价。
- 全场、自由端、柔度和域外拒绝指标。
- 阶段状态、原子产物和恢复规则。

### 15.2 FEniCSx 求解器测试

在独立求解环境中运行：

- 平面应力本构和制造解收敛。
- 左端固支和边界标记。
- 截断高斯载荷归一化。
- 总反力与总载荷平衡。
- 载荷倍增线性关系。
- FEM 到规则观测点的插值形状与精度。
- 小网格微型数据集和完整数据清单生成。

### 15.3 集成测试

- 固定外部求解器适配器和失败退出码。
- 缺失 FEniCSx 时训练不会启动。
- 求解器门禁未通过时训练不会启动。
- POD-RBF 和 DeepONet 使用相同数据身份和划分。
- 归一化器只拟合训练集。
- 验证集控制选模，封存测试不参与训练。
- 损坏清单、错误摘要和不兼容协议被拒绝。
- 已读取封存测试的运行不能覆盖后继续调参。

### 15.4 端到端测试

- 使用小网格、极少样本和短训练运行完整跨环境闭环。
- 在新进程加载已验收的微型模型并执行规则网格和任意点预测。
- 域外输入失败且退出码稳定。
- 现有标量和一维热传导闭环的全部测试继续通过。

日常 `uv run pytest` 不依赖 FEniCSx。求解器测试和跨环境端到端测试使用独立、显式命令执行；Full 科学验收不属于日常测试。

## 16. 环境方案

### 16.1 训练环境

保持当前仓库的 Python 3.11、uv 和 PyTorch 可选依赖：

```powershell
uv sync --extra operator --dev
```

该环境负责模型训练和推理，不安装 `dolfinx`。

### 16.2 求解环境

使用 Windows 原生 Miniforge/Conda 建立独立 FEniCSx 0.11 环境，环境定义提交为：

```text
environments/fenicsx-0.11.yml
```

该文件锁定主要包版本和渠道，包含 Windows 原生求解所需的 PyAMG 和 SciPy。实际解析后的完整包列表、平台信息和版本输出进入每次数据清单。

FEniCSx 官方把 Windows Conda 包标记为 beta；Windows 包不提供 PETSc/petsc4py，并要求本机具备可用的 Microsoft Visual Studio C++ 工具链以支持即时编译。因此第一版明确采用串行 PyAMG/SciPy 路线，而不编写在本机无法兑现的 PETSc 配置。若环境校准证明 Windows beta 包不稳定，则停止实施并把 WSL2/Ubuntu + PETSc 作为需要重新确认的后备架构，不能在运行中静默切换平台。

第一版使用单机 CPU 和串行或本机 MPI 求解，不启用集群、多节点或 GPU 有限元求解。环境配置完成后，`doctor` 必须验证：

1. 两个 Python 环境互相独立。
2. `dolfinx`、UFL、PyAMG、SciPy 和 MPI 可以导入，并确认未错误依赖 petsc4py。
3. 固定微型线弹性算例可以组装和求解。
4. JSON/NPZ 文件能够被训练环境读取并通过摘要验证。
5. 当前已有 91 项测试及后续新增测试仍能在 uv 环境运行。
