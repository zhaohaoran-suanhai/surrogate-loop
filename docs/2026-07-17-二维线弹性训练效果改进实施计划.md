# 二维线弹性训练效果改进实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**目标：** 在不新增 FEniCSx 样本的前提下，实现 `directional_linear_v2`、审计式复用现有 Smoke 数据，并完成与原 Smoke 同数据、同预算、同种子的第一阶段对照实验。

**架构：** 通用 `VectorDeepONet` 继续承担笛卡尔积编码，但悬臂梁实例改为输出水平、竖向两个二维响应基，再由固定物理约束层按 `cos(theta)`、`sin(theta)` 精确叠加。训练流水线通过新 `--reuse-data-from` 参数复制并验证旧 Smoke 的求解器数据，以新运行身份保存模型开发结果，原运行目录保持不变。

**技术栈：** Python 3.11、uv、Pydantic 2、NumPy、PyTorch 2.9、pytest、Ruff、现有 JSON/NPZ/SHA-256 产物协议。

## 全局约束

- 只实施设计规格的阶段 B：方向线性架构、Smoke 数据复用、方向分组报告和同数据实验。
- 本计划不实现尾部风险损失、镜像增强或方向平衡 POD；只有阶段 B 结果未达目标时才另写后续计划。
- 不启动二维线弹性 Full，不读取或消费任何 Full 封存测试。
- 不放宽 Full 的 `3%/8%/15%`、自由端 `8%`、柔度 `10%` 和 CPU 加速 `100` 倍门槛。
- 复用入口只接受 Smoke，任何身份或摘要错误必须失败，不能静默调用 FEniCSx。
- 原运行 `runs/elasticity-smoke-b694306dfbc8/` 只读，不覆盖其状态、报告、图像或数据。
- 归一化统计只由训练集拟合，开发测试只用于阶段 B 完成后的开发评价。
- 每项生产代码修改前必须先运行对应失败测试并确认因缺少目标行为而失败。
- 每项任务结束运行目标测试；最终运行 Ruff、普通 pytest、FEniCSx 求解器测试和真实微型跨环境 E2E。
- 用户可读文档使用中文；代码标识符、JSON 字段和第三方接口使用英文。

```python
if raw.ndim != 3 or raw.shape[2] != 4:

horizontal = raw[..., :2]
vertical = raw[..., 2:]
angle = physical_parameters[:, 3]
directional = (
    torch.cos(angle)[:, None, None] * horizontal
    + torch.sin(angle)[:, None, None] * vertical
)
scale = (physical_parameters[:, 2] / physical_parameters[:, 0])[:, None, None]
clamp = (physical_coordinates[:, 0] / LENGTH)[None, :, None]
```

```python
def _verify_source_request(source_run_dir: Path) -> str:
    path = source_run_dir / "request.json"
    payload = json.loads(path.read_text(encoding="utf-8"))
    if "identity_sha256" not in payload:
        raise RuntimeError("复用源请求身份字段无效")
    identity = {name: value for name, value in payload.items() if name != "identity_sha256"}
    allowed = (
        {"request", "spec"},
        {"request", "spec", "reuse_data_from", "reuse_manifest_sha256"},
    )
    if set(identity) not in allowed:
        raise RuntimeError("复用源请求身份字段无效")
    canonical = json.dumps(
        identity,
        ensure_ascii=False,
        sort_keys=True,
        separators=(",", ":"),
    ).encode("utf-8")
    digest = hashlib.sha256(canonical).hexdigest()
    if payload["identity_sha256"] != digest or payload["spec"].get("mode") != "smoke":
        raise RuntimeError("复用源请求身份无效或不是 Smoke")
    return sha256_file(path)
```

```python
def _copy_verified_file(source: Path, target: Path, expected_sha256: str) -> None:
    if sha256_file(source) != expected_sha256:
        raise RuntimeError("复用源文件 SHA-256 校验失败")
    target.parent.mkdir(parents=True, exist_ok=True)
    temporary = target.with_suffix(target.suffix + ".reuse.tmp")
    try:
        shutil.copyfile(source, temporary)
        if sha256_file(temporary) != expected_sha256:
            raise RuntimeError("复用目标文件 SHA-256 校验失败")
        os.replace(temporary, target)
    finally:
        if temporary.exists():
            temporary.unlink()
```

复用流程必须：

1. 要求 `spec.mode == "smoke"`；
2. 要求源 `status.json` 精确为 `{"state":"trained"}`；
3. 调用 `_verify_source_request`；
4. 用目标 `spec` 和 `sample_plan` 对源 manifest 调用 `_validate_manifest`；
5. 复制 `development.npz`、`sealed_test.npz`、`dataset_manifest.json`、`solver_quality.json`；
6. 对目标 manifest 再调用 `_validate_manifest`；
7. 写入 `dataset_reuse.json`：

```python

    "schema_version": 1,
    "source_run_dir": str(source_run_dir.resolve()),
    "source_request_sha256": source_request_sha256,
    "source_manifest_sha256": sha256_file(source_manifest),
    "development_sha256": files.development_sha256,
```

```python
def _case_relative_l2(
    reference: NDArray[np.float64], prediction: NDArray[np.float64]
) -> NDArray[np.float64]:
    difference = (prediction - reference).reshape(reference.shape[0], -1)
    target = reference.reshape(reference.shape[0], -1)
    denominator = np.maximum(np.linalg.norm(target, axis=1), 1e-12)
    return np.linalg.norm(difference, axis=1) / denominator
```
