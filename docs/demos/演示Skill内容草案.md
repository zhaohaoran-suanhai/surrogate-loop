# surrogate-loop 演示 Skill 内容草案

> 本文件不是可安装 Skill，只定义未来 Skill 的稳定行为。当前阶段不创建 `SKILL.md`，不决定个人安装、仓库内分发或插件形式。

## 1. 触发场景

未来 Skill 应在用户出现以下意图时触发：

- 介绍 surrogate-loop 的用途、价值或当前能力；
- 演示二维线弹性悬臂梁代理模型闭环；
- 从头运行标量、一维热传导或二维线弹性已有闭环；
- 诊断环境、求解器、训练、验收或推理问题；
- 基于仓库流程扩展新的 PDE 代理模型。

## 2. 不触发场景

- 只做与本仓库无关的通用神经网络问答；
- 只要求阅读某篇论文且不涉及仓库闭环；
- 试图让运行时执行任意方程、任意 Python 或未注册插件；
- 要求绕过摘要、状态、输入域或 Full 授权边界；
- 已有更具体的仓库任务规则明确覆盖当前工作。

不触发不等于拒绝正常协作；Agent 仍按仓库规则回答，只是不套用演示剧本。

## 3. 最小读取顺序

未来 Skill 不携带仓库状态副本，而是依次读取：

1. `AGENTS.md`
2. `docs/README.md`
3. `docs/当前能力与状态.md`
4. `git status --short` 与最近提交
5. 当前任务对应的演示手册、操作指南、配置或运行报告

只有开发或审查具体模块时才读取对应设计规格和实施计划。不要默认加载全部历史计划。

## 4. 四类任务路由

| 路由 | 识别信号 | 首要动作 |
|---|---|---|
| 演示 | 介绍、展示、讲价值 | 默认选择快速展示，确认受众与时间 |
| 运行已有闭环 | 运行、训练、复现、从头开始 | 选择白名单配置，说明成本和证据等级 |
| 诊断 | 失败、报错、环境、指标异常 | 先读状态与错误证据，再定位层级 |
| 开发新 PDE | 新方程、新几何、新输出、新求解器 | 先形成中文规格并评审，不直接写代码 |

混合请求先完成只读接管，再声明采用的主路由和后续切换条件。

## 5. 二维线弹性六阶段演示

未来 Skill 复用[二维线弹性演示手册](二维线弹性演示手册.md)的六阶段：

1. 问题与配置；
2. 环境与校验；
3. 数值求解并生成数据；
4. 训练神经算子；
5. 评价与可视化；
6. 价值与边界。

快速展示也按六阶段讲解，只是不重复计算。从头运行必须在每个阶段报告实际进度；阶段复用时明确复用身份和摘要验证。

## 6. 公式讲解合同

公式讲解至少包含：

- 平衡方程 \(-\nabla\cdot\boldsymbol\sigma=\mathbf0\)；
- 小应变与平面应力本构；
- 固支边界和参数化载荷；
- 代理映射 \(\mathcal G:(E,\nu,P,\theta,y_0,w;x,y)\mapsto(u_x,u_y)\)；
- \((P/E)(x/L)\) 硬约束的物理意义。

先用通俗语言解释“为什么需要这条公式”，再给符号形式；不把 FEniCSx 应力诊断说成神经网络正式输出。

## 7. 进度播报合同

耗时或多阶段工作统一报告：

```text
进度：n/6 阶段名称
已完成：真实完成或已验证复用的内容。
正在执行：当前动作及目的。
当前证据：已经证明什么、还没有证明什么。
下一步：后续动作、成本或授权需求。
```

Agent 不得用计划中的预期结果冒充已经完成的结果，也不得让用户在长时间运行中看不到当前阶段。

## 8. Smoke/Full 授权合同

- 假求解 E2E、真实求解器 E2E、calibration、Smoke 和 Full 必须准确命名。
- Smoke 是开发证据，允许用于发现和修正问题。
- Full 是冻结合同后的一次性确认性验收，不能由 Smoke 自动触发。
- 二维线弹性当前未完成 Full；未来 Skill 默认只能推荐快速展示或 Smoke。
- 启动 Full、改变 Full 门槛或消费封存测试必须重新获得用户明确确认。

## 9. 失败降级

- 本地运行产物缺失：引用已提交状态摘要，明确图片和报告未在本地，不伪造路径。
- 求解环境不可用：降级为快速展示，说明真实现场求解未执行。
- 时间不足：从头运行切换为快速展示，说明模式变化。
- 物理门禁失败：停止训练并诊断，不能删除失败样本。
- 训练或验收失败：保存真实失败状态，不降低门槛或绕过保护。

## 10. 完成报告合同

完成报告应先给结论，然后包含：

- 最终状态和证据等级；
- 面向受众的关键指标解释；
- 实际配置、运行目录、报告和图像路径；
- Smoke/Full、域内/域外、网络输出/求解器诊断的限制；
- 可选下一步、预计成本和授权要求。

如果只是快速展示，要明确“没有在本次对话重新训练”；如果发生阶段复用，要列出复用阶段。

## 11. 禁止复制的动态事实

未来 Skill 中禁止固化以下内容：

- 固定实测数值，包括误差、加速比、耗时、内存、显存和测试数量；
- 当前运行 ID、`runs/` 目录和具体产物是否存在；
- 可能变化的 CLI 细节、配置字段、依赖版本和设备状态；
- 某个闭环当前是否已通过 Smoke 或 Full。

这些动态事实必须从 `docs/当前能力与状态.md`、对应操作指南、当前配置和本次运行报告读取。Skill 只保存读取顺序、任务路由、解释框架和授权合同。

## 12. 未来形式评估问题

文档路由完成真实新对话演练后，再评估：

1. Skill 是只服务本仓库，还是支持多个代理模型仓库？
2. 应采用仓库内模板、个人安装 Skill，还是插件分发？
3. 如何可靠发现仓库根目录和动态文档？
4. 如何验证触发准确性、最小读取量、进度播报和 Full 授权？
5. 哪些内容属于稳定 Skill，哪些必须继续留在仓库文档？

只有这些问题有实际演练证据后，才进入正式 Skill 设计和安装。
