# 演示入口

本目录把仓库能力组织成可在新对话中直接执行的演示。默认先展示已有证据，不把现场重新训练当作理解价值的前提。

## 推荐主线：二维线弹性悬臂梁

[第一期代理模型训练闭环周报](../周报/2026-07-17-第01期-代理模型训练闭环周报.md)是管理层快速理解项目、成果和能力边界的第一入口。[二维线弹性演示手册](二维线弹性演示手册.md)用于继续展开技术演示，展示完整主线：

```text
自然语言需求
→ 白名单方程与配置
→ FEniCSx 生成二维位移数据
→ Vector DeepONet 训练
→ 冻结模型与一次性封存验收
→ accepted 推理价值和证据边界
```

它兼顾通俗解释、明确公式、六阶段进度和可复现命令。当前二维线弹性最高证据是运行 `elasticity-full-ba8ff8e584d9` 的 accepted Full；开始前先读[当前能力与状态](../当前能力与状态.md)。

## 补充：一维热传导 accepted Full 推理

一维热传导已有 `accepted` Full，可用于补充说明“通过确认性验收的模型如何经摘要校验后执行单点或完整场推理”。操作见[一维热传导闭环操作指南](../guides/一维热传导闭环操作指南.md)。

## 备用：标量 ODE 极速闭环

标量 ODE 只预测 `gamma → u(1)`，适合时间很短、没有 GPU 或只想解释闭环骨架的场景。操作见[标量闭环操作指南](../guides/标量闭环操作指南.md)。

## 选择快速展示还是从头运行

- **快速展示**：默认选择。先读第一期周报；若当前工作区保留匹配的 `runs/elasticity-full-ba8ff8e584d9/`，再展示 `acceptance.json`、严格报告和 accepted 点预测。无需重新训练。
- **从头运行**：用于现场证明真实 FEniCSx 数据生成和训练链路。运行二维线弹性 Smoke，成本估计以状态页和校准报告为准。

全新克隆不会包含被 Git 忽略的 `runs/`。此时可以准确介绍周报和稳定文档中已提交的 Full 摘要，也可以在用户接受本机成本后重新运行 Smoke；不能声称本地报告或图片已经存在。重新执行 Full、创建新 Full 身份或消费新的 sealed-test 仍需明确授权。

## 演示前检查

1. 读取第一期周报和[当前能力与状态](../当前能力与状态.md)，确认最高证据和当前推荐运行目录。
2. 询问用户更重视“快速理解价值”还是“现场从头证明流程”；未指定时选快速展示。
3. 快速展示先检查标准产物是否存在；不存在则使用已提交摘要并明确降级。
4. 从头运行先执行环境诊断与配置校验，并说明时间、设备、输出目录和 Smoke 语义。
5. 已有 accepted Full 可用于只读报告和域内推理；未经新的明确确认，不重新训练二维线弹性 Full。
