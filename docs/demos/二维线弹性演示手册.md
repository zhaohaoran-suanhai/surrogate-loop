# 二维线弹性悬臂梁演示手册

## 1. 演示价值与受众

这条演示回答一个具体问题：当材料、载荷和载荷位置变化时，能否先用有限元生成可信样本，再训练神经算子快速给出整片悬臂梁的二维位移场？

它适合不了解神经网络训练的受众，因为每一步都有可见输入、输出和证据；也适合技术受众，因为方程、数据来源、模型、评价和可信边界都有明确合同。

演示价值不等于宣称模型已经达到工程认证精度。当前流程已经完成 calibration、Smoke 和一次 Full 确认性验收；运行 `elasticity-full-ba8ff8e584d9` 状态为 `accepted`，但该结论仍只覆盖冻结的悬臂梁模板、参数域与验收合同。

## 2. 一分钟通俗介绍

传统有限元每换一组材料或载荷都要重新组装和求解方程。这里先让 FEniCSx 对一批代表性参数做可靠计算，得到位移场标签；Vector DeepONet 再学习“参数和查询坐标到位移”的映射。训练完成后，同一训练域内的新参数可以通过一次神经网络前向计算快速得到位移场。

整个仓库不是让大模型随意写方程并执行。Agent 负责把自然语言需求收敛为人能检查的白名单配置，固定 Python CLI 负责数值求解、训练、验收、摘要和推理保护。

## 3. 控制方程、本构和代理映射

在二维区域 \(\Omega=[0,L]\times[0,H]\) 上，忽略体力时的静力平衡方程为

\[
-\nabla\cdot\boldsymbol{\sigma}(\mathbf u)=\mathbf 0
\quad \text{in }\Omega.
\]

小应变定义为

\[
\boldsymbol{\varepsilon}(\mathbf u)
=\frac{1}{2}\left(\nabla\mathbf u+\nabla\mathbf u^{\mathsf T}\right).
\]

平面应力本构写成

\[
\boldsymbol{\sigma}
=2\mu\boldsymbol{\varepsilon}
+\lambda_{\mathrm{ps}}\operatorname{tr}(\boldsymbol{\varepsilon})\mathbf I,
\qquad
\mu=\frac{E}{2(1+\nu)},
\qquad
\lambda_{\mathrm{ps}}=\frac{E\nu}{1-\nu^2}.
\]

左端固支：\(\mathbf u=\mathbf 0\)；右端满足 \(\boldsymbol{\sigma}\mathbf n=\mathbf t(P,\theta,y_0,w)\)，其中载荷采用受约束的截断高斯分布。

神经算子学习

\[
\mathcal G:(E,\nu,P,\theta,y_0,w; x,y)
\longmapsto (u_x(x,y),u_y(x,y)).
\]

实现利用线弹性比例关系和固支边界，把网络输出写成含

\[
\frac{P}{E}\frac{x}{L}
\]

的硬约束形式。这样 \(x=0\) 时预测位移自动为零，并把主要幅值缩放从网络学习中分离出来。

## 4. 六阶段演示流程

1. **问题与配置**：把自然语言需求对应到固定悬臂梁模板、参数域和受审查配置。
2. **环境与校验**：检查训练环境、FEniCSx 求解环境和 JSON 合同。
3. **数值求解并生成数据**：FEniCSx 对参数样本求解位移，执行残差、力平衡等门禁，写出版本化数据与摘要。
4. **训练神经算子**：POD-RBF 提供诊断基线，Vector DeepONet 学习参数和坐标到二维位移。
5. **评价与可视化**：Smoke 在开发测试集诊断；Full 冻结后一次性读取 sealed-test，计算场误差、自由端、柔度、固支边界和加速比。
6. **价值与边界**：解释加速比、最终状态、Smoke/Full 区别和可信推理限制。

快速展示读取已有证据并依次讲解这六段；从头运行则实际执行第 2—5 段，并持续报告进度。

## 6. 从头运行模式

从头运行默认使用 Smoke，不触及 Full。操作前先说明输出写入 `runs/`、会调用独立 FEniCSx 环境、耗时估计来自状态页，并确认电源和计算资源可用。

```powershell
uv run surrogate-loop elasticity2d doctor
uv run surrogate-loop elasticity2d validate --config examples/elasticity_2d_cantilever/smoke.json
uv run surrogate-loop elasticity2d run --config examples/elasticity_2d_cantilever/smoke.json --runs-dir runs --request "演示：从头训练二维悬臂梁位移场代理模型"
uv run surrogate-loop elasticity2d report --run-dir runs/<命令返回的运行目录>
```

不要猜测运行目录；以 `run` 返回的 JSON 为准。若命令恢复了已完成阶段，说明阶段身份与摘要如何匹配，并把本次称为“复用后继续”，不能称为完全从零。

校准是可选的前置科学检查。需要重新证明网格、物理门禁和本机成本时，先读操作指南并执行 calibration；额外成本从状态页的最近记录估算。

## 8. 指标如何解释

- **全场相对 L2**：整片位移场总体形状和幅值的相对误差；中位数表示典型样本，P95 与最差值暴露尾部风险。
- **自由端误差**：关注悬臂梁最直观、通常位移最大的区域。
- **柔度误差**：检查载荷做功相关的整体结构响应。
- **固支边界误差**：验证左端零位移硬约束是否生效。
- **加速比**：必须采用单个 FEM 样本与单个神经网络样本的一致 CPU 口径；它回答“推理多快”，不回答“预测是否足够准”。

当前 Full 数值只从状态页和 accepted 运行的 `acceptance.json` 读取：全场相对 L2 中位/P95/最差为 `0.2519%/1.5152%/4.4492%`，自由端 P95 为 `0.7550%`，柔度 P95 为 `3.2833%`，固支边界误差为 `0`，CPU 加速比为 `931.18×`。面向普通受众时先说“典型表现、尾部风险、速度价值”，再按需展示数字。

## 9. 证据等级与 Full 边界

- 假求解 E2E 只证明编排。
- 真实 FEniCSx 微型 E2E 证明跨环境链路。
- calibration 证明数值、物理门禁和成本基线。
- Smoke 使用开发测试集，能指导调整，因此只形成开发证据。
- Full 使用冻结合同和一次性封存测试，才可能形成确认性 `accepted` 结论。

二维线弹性 Smoke 的状态仍是 `development_complete`，用于追溯开发和同数据架构改进。最高证据已是 `elasticity-full-ba8ff8e584d9` 的 accepted Full；该运行可以只读报告并执行域内可信推理。现场不重跑 Full；重新训练、新建 Full 身份或消费新的 sealed-test 必须重新获得用户明确确认。

## 10. 故障降级

- FEniCSx 环境不可用：切换为已提交摘要的快速展示，明确现场求解未执行。
- 本地 `runs/` 缺失：不展示不存在的图片；使用状态页摘要或在用户接受成本后重新跑 Smoke。
- 时间不足：从头运行切换为快速展示，并明确发生了模式切换。
- 物理门禁失败：停止训练并查看求解器诊断，不删除失败样本。
- 训练未达指标：保留 `rejected` 或 `development_complete` 诊断，不降低 Full 门槛。
- 阶段复用：说明复用身份和摘要验证，不能描述为完全从零运行。
