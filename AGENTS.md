# 仓库工作规则

## 目标

本仓库构建一个自然语言驱动、结构化约束、确定性验证、全流程可审计的代理模型训练最小闭环。

## 目录边界

- `docs/`：中文设计、计划、架构和操作指南。
- `src/surrogate_loop/`：可安装、可测试的核心代码。
- `examples/`：可复现输入配置和中文算例说明，不复制核心代码。
- `tests/`：按 `unit`、`integration`、`e2e` 分层的自动化测试。
- `runs/`：本地运行产物，不提交生成文件。

## 强制规则

- 使用 Python 3.11 和 uv，不直接向系统 Python 安装依赖。
- 功能和修复先写失败测试，再写最小实现。
- 归一化器只能在训练集上拟合；测试集只用于最终模型的一次评估。
- 禁止 `eval`、`exec`、任意方程执行和加载外部不可信模型文件。
- 用户可读文档使用中文；代码标识符、JSON 字段和第三方库接口使用英文。
- 不把核心实现写入 `examples/` 或 `tests/`。

## 提交前检查

```powershell
uv run ruff check .
uv run pytest
```

## 新对话最小阅读顺序

1. 读取本文件。
2. 读取 `docs/README.md`。
3. 读取 `docs/当前能力与状态.md`。
4. 查看 `git status --short` 和最近提交，确认工作区状态。
5. 将任务分类为演示、运行已有闭环、诊断或开发新 PDE，再按文档地图按需读取。

不要默认全文读取所有实施计划。只有设计、修改或审查具体闭环时，才读取相应规格和历史计划。

## 证据与授权

- 普通假求解 E2E 只证明编排、保护机制和产物兼容性。
- 真实求解器 E2E 证明跨环境链路；calibration 证明数值与物理门禁；Smoke 是开发证据；Full 是一次性确认性验收。
- 二维线弹性当前完成 calibration 和 Smoke，未完成 Full，不能表述为已通过确认性精度验收。
- 未经用户新的明确确认，不启动 Full、不放宽门槛、不绕过摘要、不把失败样本静默删除。
- 从头运行发生恢复或阶段复用时，必须明确说明复用了什么，不能称为完全从零运行。
